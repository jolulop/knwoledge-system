# Context

This document defines the shared language for the Knowledge System project.

## Core Terms

**Raw Source**
An original document, transcript, image, screenshot, HTML file, PDF, DOCX, spreadsheet, or Markdown file stored as the source of truth. Raw sources are not modified by agents.

**Source ID**
A deterministic identifier for a raw source derived from its content: `src_<first 16 hex characters of the SHA256>`. Because it is content-derived, byte-identical files share a single Source ID, which makes repeated intake scans idempotent.

**Manifest**
A JSON record at `raw/manifests/<source_id>.json` describing one unique raw source. There is one manifest per unique content, not per file. It holds the checksum, canonical file metadata, and an `occurrences[]` list. In Phase 1 and Phase 2 the manifest files are the authoritative local runtime listing for the `/sources` endpoint and CLI, but generated `*.json` manifests are not committed to git.

**Occurrence**
One observed file path for a given Source ID. Byte-identical copies of the same content are recorded as additional occurrences inside the one manifest rather than as separate manifests. A redundant copy is an exact (SHA256) duplicate and does not require human review.

**Raw Path**
The manifest may retain an absolute `raw_path` for local runtime operations where resolving the exact file matters. Portable references, API responses, wiki pages, and normalized artifacts should use repository-relative paths such as `relative_raw_path` and `occurrences[].relative_path`.

**Normalized Document**
A parsed representation of a raw source, usually Markdown or JSON, used for indexing, chunking, citation anchoring, and downstream processing. Normalized artifacts are content-keyed: each raw source's Markdown, chunks, tables, and extraction log live in per-source files named by its Source ID, mirroring the manifest model.

**Chunk**
A contiguous span of a normalized document produced by heading-aware splitting, identified as `<source_id>::<ordinal>` and stored per source at `normalized/chunks/<source_id>.jsonl`. Each chunk carries citation anchors so downstream answers can cite verifiable evidence.

**Citation Anchor**
A mechanically derived pointer to where evidence lives in a source ŌĆö heading/section path and character range (always), source page number (for paginated formats), or table/sheet reference. Anchors are drawn only from the accepted anchors in `policies/citation.yaml` and are never estimated or invented.

**Structured Citation**
The typed object a claim, synthesis, or query answer uses to cite evidence (ADR-0020). Its authoritative locator is `(source_id, char_start, char_end)` ŌĆö a character range into the source's normalized Markdown ŌĆö with optional `page`/`page_end`, `section`, and `table_reference`/`sheet_reference`, plus an advisory, non-authoritative `chunk_id`. Free-text "location" strings are not citations.

**Node Identifier**
The stable id every wiki node carries: `source_id` (`src_ŌĆ”`, content hash) for sources, and for semantic nodes a creation-time hash frozen in frontmatter ŌĆö `item_id` (`itm_ŌĆ”`, ADR-0059), `claim_id` (`clm_ŌĆ”`), `synthesis_id` (`syn_ŌĆ”`), `query_id` (`qry_ŌĆ”`). The id is permanent across renames; the human-facing slug/title may change (ADR-0021). The item id is **type-neutral**: classification (`item_type`) is governed metadata, never part of identity ŌĆö retype changes no id. The pre-restart typed prefixes (`cpt_/ent_/per_/org_/prj_`, ADR-0017/0021) are superseded by ADR-0059 and exist only in historical ADRs/backups.

**Extraction Log**
A per-source diagnostic record at `normalized/extraction_logs/<source_id>.json` describing one extraction run: tool and version, character counts, warnings such as `needs_ocr`, and any error or skip reason. From ADR-0054 on it also records the integer `extract_code_version` ŌĆö extractor *implementation* provenance (distinct from the tool's own version); a missing field means "pre-marker / older extractor". Observability only: nothing gates on it. The manifest holds the summary extraction status; the extraction log holds the detail.

**Wiki Page**
A derived Markdown page used for human browsing and agent navigation. Wiki pages are generated or maintained from raw sources and normalized documents. The entire `wiki/` layer ŌĆö including `index.md` and `log.md` ŌĆö is mutable local runtime data, not code: it is gitignored, regenerated by the pipeline, and made durable by the backup mechanism rather than by git (ADR-0014).

**Source Page**
A wiki page representing one raw source, stored content-keyed at `wiki/Sources/<source_id>.md` with the human-readable title carried in frontmatter (ADR-0015). In the Phase 3 deterministic backbone a Source page asserts only mechanically-derived facts; sections that require an LLM (tags, concepts, entities, claims, etc.) are rendered as `_Pending semantic enrichment_` placeholders until a later phase fills them (ADR-0013, ADR-0016).

**Input Fingerprint**
A hash recorded in a generated page's frontmatter (`input_fingerprint`) covering every input that determines the page ŌĆö for deterministic backbone pages: schema version, template, the source's normalized Markdown, manifest fields, and config; for enriched pages it also covers the prompt/template version and the resolved `model_ref` (provider + model id). Regeneration skips a page when the candidate's fingerprint matches the stored one, so template/config/extractor/model changes are picked up without `--force`. Deterministic artifacts (Source pages, `index.md`) carry no wall-clock timestamp; freshness lives in the fingerprint, file mtime, `log.md`, and the job record (ADR-0023, ADR-0027).

**Enrichment**
The Phase 3.5 LLM stage that fills the semantic layer the deterministic backbone leaves as placeholders: real summaries, tags, concepts, entities, claims, and synthesis. Enrichment treats source text as untrusted data and the model as a pure textŌåÆdata function with no acting tools; its output is schema-constrained, citation-grounded, and human-reviewed for semantic/destructive changes (ADR-0025, ADR-0026). It is provider-agnostic ŌĆö any model reachable through an LLM Adapter can serve a pass. Enriched pages carry `generation_status: enriched`.

**Enrichment Artifact**
A per-source, content-keyed record of an enrichment pass's validated, schema-conformant output (e.g. `normalized/enrichment/<source_id>.json`). It is the artifact of record for LLM output; enrichment passes write it, never `wiki/Sources/<source_id>.md` directly. The deterministic Source-page renderer stays the sole writer of the `.md` and composes the artifact into the page, filling the `_Pending semantic enrichment._` placeholders when present and falling back to them when absent ŌĆö so the page remains a pure function of its inputs and a full rewrite never clobbers LLM content. The artifact carries its own `input_fingerprint` (normalized Markdown + prompt/template version + `model_ref` + schema) that drives re-enrichment (ADR-0025, ADR-0027). It is derived state ŌĆö regenerable by replaying the [[response-cache]] (free, no provider call) when the normalized text reproduces ŌĆö and is not itself backed up; the cache is the durable record of LLM output (ADR-0027).

**LLM Adapter**
The thin, provider-agnostic seam that abstracts model connection and usage behind one uniform call: `LLMClient.parse(messages, schema, model_ref) -> validated object`, returning a schema-valid object or raising a typed error. Per-provider adapters implement it ŌĆö native `anthropic` SDK, native `openai` SDK, and a local adapter over the OpenAI-compatible HTTP wire (Ollama, vLLM). Provider-specific accelerators (prompt caching, Batch API) are optional adapter capabilities advertised by flags like `supports_batch`, never guarantees the caller depends on (ADR-0025).

**Model Reference**
The provider-aware identifier a tier maps to, written `provider:model_id` (e.g. `anthropic:claude-haiku-4-5`, `openai:ŌĆ”`, `local:ŌĆ”`). The provider half selects the LLM Adapter and its connection config (base URL, API-key env var); a `local` reference additionally needs a base URL. `model_ref` is what the response-cache key and `input_fingerprint` record, so switching provider or model is a cache miss and a refresh, not a silent collision (ADR-0025, ADR-0027).

**Model Tier**
The configurable model routing used by enrichment: tier 1 (light, default `anthropic:claude-haiku-4-5`) for summaries/tags, tier 2 (standard, default `anthropic:claude-sonnet-4-6`) for concept/entity/claim extraction, tier 3 (heavy, default `anthropic:claude-opus-4-8`) for synthesis/contradiction. Each pass declares its tier in code; only the tierŌåÆ`model_ref` mapping is config, and it may point all tiers at one model or mix providers (ADR-0025).

**Response Cache**
A persistent local cache of raw LLM responses keyed by `hash(prompt + model_ref + schema)` ŌĆö `model_ref` includes the provider so the same logical model served via two providers does not collide ŌĆö stored under `db/` and covered by backups. Because LLM output is not byte-reproducible, the cache lets a rebuild or forced re-render replay the stored response ŌĆö reproducible and free ŌĆö until an input changes or the cache is busted (ADR-0027).

**Summary Stub**
A deterministic, extractive `> [!summary]` callout written without an LLM ŌĆö typically the first meaningful paragraph of the normalized Markdown (or a structural fallback when text is too sparse). It is marked `summary_status: stub` in frontmatter; a later phase replaces it with an LLM summary and flips the flag to `enriched`. The maintenance linter treats a stub as expected, not as summary rot (ADR-0016).

**Knowledge Item (Item Page)**
The semantic node family that replaced Concept/Entity/Person/Organization/Project pages (ADR-0059): one structural type (`node_type: item`), classified by **knowledge-object role** via the 15-value [[knowledge-item-taxonomy]]. Item pages live in one flat directory, slug-keyed by canonical name (`wiki/Items/<slug>.md`), and carry a stable type-neutral `item_id` (`itm_ŌĆ”`) in frontmatter that the graph keys edges on, plus `item_type` and an `aliases` list ŌĆö so **renames** re-slug + relink while edges never break (ADR-0017 mechanics carried over), **retype** is a metadata flip (no id change, no page move), and a **merge** re-points edges to the survivor + tombstones the absorbed id (ADR-0050). "Item" alone is ambiguous with "review item" ŌĆö use *knowledge item* where confusion is possible; review artifacts remain "review items". One node per canonical referent: same-name-different-referent is resolved by `split`, never by type.

**Candidate Node**
A promotable knowledge item evidenced by too few independent sources to be first-class. It is created as a `candidate`/`stub` (low confidence) and kept out of promoted navigation and synthesis. It promotes to `active` automatically once Ōēź2 *independent* sources evidence it (recurrence), or early by human review; both paths run through the single `promote_candidate_node` review action and the deterministic `promote` worker (ADR-0018, Phase 3.5b slice 5). Exception: a candidate carrying the [[unclassified-sentinel]] never auto-promotes and cannot be approved without a real `item_type` amendment (ADR-0059).

**Independent Source**
A source that counts toward a node's promotion threshold. Exact (SHA256) duplicates share one `source_id` and count once. Independence is judged from manifest provenance (`author`/`publisher`/`report_family`/`canonical_url`): two sources are independent iff Ōēź1 *comparable* key (present on both) differs and no comparable key is equal, after canonicalization (whitespace/case; URLs also drop fragment + trailing slash). Manifests carry no provenance by default, so the gate is conservative ŌĆö nothing auto-promotes until provenance is populated (ADR-0018).

**Claim**
An atomic factual statement derived from one or more sources. A claim cites its evidence as a Source page plus a stable mechanical anchor (page / section / character range into the normalized Markdown), never a chunk ordinal; a `chunk_id` may be kept only as a non-authoritative pointer (ADR-0019). Claims require citations where possible.

**Synthesis**
A higher-level explanation, comparison, or conclusion across multiple sources, concepts, or claims, stored as a `synthesis` node (`syn_ŌĆ”`) page. In Phase 3.5c it is produced one-per-`active`-concept/entity, eligible only when Ōēź2 `active` claims connect to the target through the graph neighborhood and those claims are supported by Ōēź2 *independent* sources (ADR-0018/0031); candidate or single-source targets get no synthesis. It is grounded by referencing already-grounded `claim` nodes (`derived_from` synthesisŌåÆclaim), with any direct source quote run through the verbatim citation gate (unlocatable ŌåÆ dropped, or generation marked `partial`); those `derived_from` edges are `active` as **provenance, not approval**, so the node itself stays `candidate`. A synthesis page is written to disk under `wiki/Synthesis/` so a human can read it; like a candidate concept it **is listed in `index.md` marked `candidate`** (the index is the full page listing for discovery/review, not a promoted-only view), but is **excluded from promoted/authoritative use** ŌĆö never returned as evidence for any later synthesis or query answer, and never surfaced by Phase 4 retrieval as evidence ŌĆö **until promoted to `active`**. The load-bearing invariant is *evidence-eligibility, not index presence*: `candidate` (and `deprecated_candidate`) means readable and navigable but not citable. Its promotion is **review-only via a `propose_synthesis` review item ŌĆö there is no recurrence auto-promote path**, precisely because a synthesis is born from Ōēź2 sources by construction and would otherwise auto-promote and bypass review; approval flips it to `active` (mirror + audit), rejection sets `review_status: rejected` and `status: deprecated_candidate` (the lifecycle vocabulary has no `rejected` node status) (ADR-0022, ADR-0028, ADR-0031).

**Contradiction**
A review-gated `contradicts` assertion between two `claim` nodes that conflict, proposed by the Phase 3.5c tier-3 pass. It is written as one edge with the two claim ids **sorted lexically** (`src_id < dst_id`) so the symmetric relation collapses to a single row and a single `resolve_contradiction` review; born `status: proposed` (invisible until reviewed). The edge's evidence anchor (the src claim's primary active citation) is **advisory only** ŌĆö the authoritative, two-sided evidence stays the two Claim pages' structured citations, and the review item carries both claim ids plus rendered context for both sides. Contradictory claims stay visible until reviewed (Build Spec). A human resolves it with one of three outcomes: `acknowledge` (edge ŌåÆ `active`, both claims live), `supersede` (a named winner gets an `active` `supersedes` edge and the loser is deprecated to `deprecated_candidate` via the `deprecate_wiki_page` audit path, with the cause recorded in `audit_log`; the `contradicts` edge still activates), or `reject` (edge ŌåÆ `rejected`). A `supersede` decision is never silently recorded without its effects. The LLM proposes; the human disposes (ADR-0031).

**Candidate Pair (Blocking)**
The bounded set of claim pairs the Phase 3.5c contradiction pass actually sends to the heavy model, produced by a deterministic, zero-cost *blocking* step over the graph instead of infeasible all-pairs comparison: two claims are a candidate pair iff they share Ōēź1 `active` concept/entity node (co-mention via `active` `mentions` edges) **and** come from two independent sources (the ADR-0018 independence test ŌĆö a source cannot contradict itself). Each pair is fingerprint-idempotent and cache-replayed, the fingerprint covering **both claims' text and their active citation anchors plus the shared blocking node ids** (not text alone ŌĆö evidence context can change while text survives) and the prompt/model/schema; a corpus-level fingerprint is rejected. Vector-similarity blocking is a recorded but deferred second channel (ADR-0031).

**Review Item**
A proposed semantic or destructive change requiring human approval, such as deletion, entity merging, contradiction resolution, deprecation, or archiving.

**Graph (Source of Truth)**
The SQLite store under `db/graph.sqlite` that is authoritative for **relationships (edges)** ŌĆö keyed on stable typed ids (`source_id`/`concept_id`/`entity_id`/`claim_id`), not slugs, from the governed Build Spec ┬¦6.2 vocabulary (minus `needs_review`, which is represented as `status`). Each row is one *assertion* of a relationship ŌĆö distinct evidence spans and asserters coexist, and a relationship exists/projects iff it has an `active` assertion; each carries `status` (proposed|active|rejected|superseded), provenance (`asserted_by`), confidence, an evidence anchor, and a `review_id`. Node *metadata* (id/title/slug/status/aliases) stays owned by wiki frontmatter (ADR-0017/0021/0022) ŌĆö sources by their manifest (ADR-0008); the graph's `nodes` table is a derived index rebuilt from those, and the backlink sections in pages are a derived projection of the graph's `active` assertions, so rename/merge is an id-level redirect and backlinks stay synchronized by construction. An LLM- or prose-authored assertion enters as a review-gated `proposed` candidate (pending or deferred stays invisible; approved ŌåÆ active, rejected ŌåÆ rejected) and never appears as a backlink until approved (ADR-0029, ADR-0030; lands in Phase 3.5b).

**Hybrid Retrieval**
The combination of keyword search, semantic/vector search, graph traversal, and summary-first wiki navigation. The two *chunk-evidence* channels (keyword + vector) are fused into one ranked `evidence[]` by [[reciprocal-rank-fusion]]; graph and navigation stay separate `/search` result groups (ADR-0032 decision 6).

**Reciprocal Rank Fusion (RRF)**
The rank-based, scale-free way `/search` combines the keyword (BM25) and vector (distance) chunk-evidence channels into one ranked list (Phase 4e): each chunk scores `╬Ż 1/(k + rank_c)` over the channels that returned it, `k = rrf_k` (default 60, `policies/retrieval.yaml`). Deduped by the citation `(source_id, char_start, char_end)`; a hit found by both channels merges into one evidence object whose top-level `score` is the fused RRF value, `retrieval_path` lists the contributing channels, and an additive `channels` field carries each channel's 1-based `rank` + native `score` (BM25; vector distance, lower=better). Deterministic: identical query+index ŌćÆ identical order, tie-break `(source_id, ordinal, char_start)`. Weighted/score-normalized fusion and graph boosts stay **deferred and eval-gated** ŌĆö their parameters can't be justified without a relevance oracle, whose prerequisite is the [[retrieval-relevance-eval]] corpus (ADR-0032 addenda 7 + 9, ADR-0038); RRF fuses chunk evidence only. The **[[item-type-faceting]] evidence boost** (ADR-0062) is the one bounded, opt-in additive post-fusion re-rank layered on top.

**item_type faceting** (ADR-0062)
The retrieval-side payoff of the ADR-0059 [[knowledge-item-taxonomy]]: an optional, multi-value `item_type` facet (repeatable `?item_type=` on `/search`; a list field on `/query`) that steers results by knowledge-object role. **Precise where `item_type` is native, advisory where it is only bridged.** On the **navigation** channel (item-page discovery, a new `UNINDEXED` nav-index column populated only for Item pages, `INDEX_VERSION` 1ŌåÆ2) and the **graph** channel (native `nodes.item_type`), the facet is a hard `item_type IN (ŌĆ”)` filter ŌĆö but only where the result *has* an `item_type`: it is a **type predicate, not a layer filter**, so Source/Claim/Synthesis/Query results (no `item_type`) pass through untouched (item-only discovery stays the `page_type=item`/`node_type=item` levers). On the **evidence** channel (chunks; feeds `/query`), `item_type` is not a chunk property ŌĆö the only link is the coarse, **active-only** bridge chunk ŌåÆ source ŌåÆ active `mentions` ŌåÆ active item ŌåÆ `item_type` ŌĆö so it is an **advisory bounded boost, never a filter** (protects the [[citation-grounding]]/eligibility invariant: citations come only from citable chunks). The boost is additive, applied post-[[reciprocal-rank-fusion]]/pre-cap via one batched per-source lookup, policy-knobbed as `item_type_boost` (`0` disables), and **weaker than primary relevance** ŌĆö it breaks ties / nudges a few positions but must never become a hidden evidence filter (a highly-relevant off-type chunk still outranks a weak on-type one). The facet vocabulary is the **15 real `taxonomy.ITEM_TYPES`**; unknown values and the `unclassified_review_required` sentinel are **rejected with 400**. Opt-in: an absent facet changes nothing (unfaceted default ranking is byte-identical, so the ADR-0038 relevance baseline is not re-recorded). Proven in the **structural** eval lane only. Deferred alternative: **item-seeded retrieval** (facet selects items ŌåÆ their sources scope evidence as a real filter) ŌĆö a future retrieval mode, revisited only if the advisory boost proves insufficient.

**Structural vs relevance retrieval eval** (ADR-0032 addendum 8, ADR-0038)
Two distinct retrieval evals, never conflated. The **structural retrieval eval** (`tests/test_retrieval_evals.py` + `evals/golden_retrieval.yaml`) runs over a programmatic fixture with a deterministic **fake embedder** and asserts *structure* ŌĆö anchors, status-awareness, router taxonomy, RRF shape/order, retention. It is **key-free and the CI gate**. The **retrieval relevance eval** (ADR-0038: `evals/corpus/` + `evals/golden_retrieval_relevance.yaml` + an opt-in `scripts/eval_retrieval.py`) measures *semantic relevance* ŌĆö does `run_search` surface the right citable chunks/sources for a query, by source-level binary judgments scored as recall@k / MRR / success@k. It needs the **real embedder** (no LLM key), so it is **opt-in/manual, never a CI gate**, and produces a reproducibility-stamped report (`embedding_model_ref`, index version, `rrf_k`, `graph_mode=empty`). It is the prerequisite that unblocks weighted RRF / graph-boost tuning, and is distinct from the deferred *answer-quality* eval (`/evals/run`, ADR-0036). v1 cases are **source-level** (relevant/irrelevant by filename); the **multi-chunk extension** (ADR-0038 ┬¦Multi-chunk) adds **chunk-level** cases (`category: chunk_disambiguation`) that locate a chunk by a `contains:` phrase resolved to its authoritative **citation key** `(source_id, char_start, char_end)` (not the advisory `chunk_id`), with an **intra-doc `near_miss`** chunk ŌĆö testing chunk-level *discrimination* (the right chunk ranks above the near-miss) and a **chunk-granular per-channel diagnostic**. Chunk cases are reported separately (`## Chunk-Level Aggregate` / `## Chunk Source Continuity`) and never contaminate the source-level headline; multi-chunk docs use `##` sections to force deterministic chunk boundaries. A chunk failure labelled `keyword_prefers_relevant_vector_prefers_irrelevant` would be the first real evidence that *fusion weighting* (not just a better embedder) could help.

**Answer-quality eval** (ADR-0042)
The third eval, never conflated with the two retrieval evals above: a **key-required, loopback-only, read-only operator workflow** that measures the **cited-answer pipeline** (`POST /query`) against a curated **real-vault golden Q&A corpus** and scores it **deterministically ŌĆö no LLM judge**. The LLM key is used only to *generate* answers; scoring is mechanical against per-question predicates (expected sources cited, forbidden sources not cited, abstain expected-vs-actual, `unsourced_count`/`security_rejected_count` zero, answer-when-expected, citation recall/precision over `source_id`s). A reusable core `app/workers/eval_answers.py` (fake-client testable, so **CI stays key-free**) backs two thin endpoints: **`POST /evals/run`** (explicit `confirm_cost`, `EVAL_MAX_QUESTIONS_DEFAULT` + hard-cap-clamped `limit`, 503 if the LLM is unconfigured, a key-free `dry_run` corpus pre-flight) and **`GET /evals/results`** (key-free read of stored runs). It is **read-only with respect to vault source-of-truth + semantic state** (writes nothing under `raw/`/`normalized/`/`wiki/`/`reviews/`/`raw/manifests/`/graph; always `/query` with `save:false`), while it MAY write its own result artifacts and MAY populate the LLM [[response-cache]] on a miss like the normal `/query` path (`db/llm_cache.sqlite` is an execution artifact, not vault SoT); `fresh:true` bypasses cache lookup **and** write for a clean provider-drift run, and `cache_hits`/`cache_misses` are recorded. Durable results under `evals/reports/answers/` (gitignored) store **only ids/flags/scores/metadata ŌĆö no prompt, evidence, or answer prose, and no absolute paths** (privacy: evidence packs + answer text can echo private source text; a root *fingerprint*, not `/home/...`). The real corpus `evals/golden_answers.local.yaml` is **gitignored** (operator data); a committed `evals/golden_answers.example.yaml` documents the schema. The headline is a **reproducibility-stamped snapshot, not a CI gate** (answer generation is nondeterministic). An LLM-as-judge prose-grading "analysis lane" is deferred. Distinct from the deterministic `/query` *structural* eval (`golden_questions.yaml`, fake adapter) and the [[structural-vs-relevance-retrieval-eval]] retrieval evals.

**Cited Answer (Query Answer)**
The output of `POST /query` (Phase 5): an LLM-synthesized answer over [[reciprocal-rank-fusion]] chunk evidence in which **every asserted factual claim is mechanically grounded**. The LLM returns ordered claims that reference retrieved evidence by id only (never inventing anchors); the harness builds each citation from the *retrieved* evidence and runs the verbatim [[citation-grounding]] gate (`ground_citation`, ADR-0026). Only grounded claims enter the answer body ŌĆö `max_answer_unsourced_claims: 0` applies to the asserted body; ungrounded model output is logged in an audit "Unsourced Claims" section, not asserted. Zero grounded claims ŌåÆ the answer abstains with `"No source found in vault."` Citations come **only** from citable source chunks, never `candidate` node prose (ADR-0032 eligibility invariant). Saved to `wiki/Queries/<query_id>.md` only on explicit request; a saved query is a navigable answer artifact (`type: query`, `answer_eligible: false`) with citations in frontmatter and **no graph edges/authority** (ADR-0034).

**Evidence Pack**
The delimited, **untrusted** bundle of retrieved citable chunks passed to the Phase 5 synthesis prompt: each chunk gets a stable in-request `evidence_id` plus its authoritative anchor (`source_id`, `char_start`, `char_end`) and text/quote. The LLM may reference these only by `evidence_id`; the harness ŌĆö not the model ŌĆö builds the citation objects from the pack, so an anchor can never be hallucinated or computed by the model (ADR-0026/0034).

**Untrusted-source prompt encoding** (ADR-0061)
The **container-driven** contract for how untrusted text is placed into an LLM prompt, hardening [[llm-untrusted-input-and-grounding]] (ADR-0026) against **delimiter breakout** (a source containing its builder's literal closing tag ŌĆö `</source_document>`, `</claims>` ŌĆö escaping the data block and becoming instruction-adjacent). The rule follows the container **shape**, not the pass: text in **XML/tagged blocks** is **entity-escaped** for `&`, `<`, `>` (in that order, `&` first) before interpolation; text in **JSON payloads** (the [[evidence-pack]], ADR-0034 review B1) is emitted with `json.dumps`; **no raw untrusted interpolation** anywhere. Whole Markdown documents are escaped, **not** JSON-serialized, to preserve the extraction-quality assumptions of ADR-0055/0056 (headings/tables/lists stay legible to the model). The trust boundary is a **validated structural token**, never "system-generated": every interpolated string is escaped *unless* it is a numeric token (validated/formatted) or an **ID validated against the canonical `<prefix>_[0-9a-f]{16}` grammar** ŌĆö an id containing `<`/newline/whitespace is corrupt state that **fails loudly**, not silently escaped. `title` (from the attacker-influenceable `original_filename`, interpolated outside the delimiter) is additionally **single-line-sanitized** (`\r\n\t`/control ŌåÆ space). Escaping the claims **body** collides with verbatim [[citation-grounding]], so any model output that quotes escaped source is **`html.unescape()`-d exactly once** at the grounding boundary before `locate_quote`, and the **stored citation quote is the unescaped, source-faithful form** ŌĆö the only pass affected (summary is page-level, items are abstractions, contradiction/synthesis read already-stored `claim_text`). Any prompt-encoding change **bumps the pass's `*_PROMPT_VERSION`** (fingerprint/cache refresh) even when instruction wording is unchanged. Distinct from the sibling scoping fix in the same ADR: filesystem-leak validation (`validate_wiki`) applies to **frontmatter/structured metadata only** ŌĆö Source excerpts are rendered source data and may legitimately contain path-shaped strings, so body prose is no longer scanned for `/home/ŌĆ”`, `/var/ŌĆ”`, `C:\ŌĆ”`.

**Review Ledger / Human Review UI**
The Phase 6 governance surface over the `reviews/` artifacts (ADR-0018/0035). **Decide and apply are decoupled:** recording a human decision (approve/reject/defer) only moves the item between `reviews/pending|approved|rejected/` + writes `audit_log/` (`resolve_review_item`), while the *effect* is applied later by deterministic, **key-free**, idempotent executor passes that scan `approved/` (`apply_resolved_syntheses`, `promote_candidates`, `apply_resolved_contradictions`, and the Phase-6 `apply_approved_deprecations`). The **ledger is type-complete** (every pending type is listed + decidable ŌĆö the human must see everything needing a decision), but `POST /reviews/apply` only runs the executor-backed actions and reports unapplied types/reasons honestly. The UI is **server-rendered HTML on the existing FastAPI app** over a deterministic JSON read model (`GET /reviews`), **never authority** (it calls the same review-service primitives), with mandatory proposal preview before approve; safety rests on the loopback-only no-auth bind (ADR-0009).

**Governance executor risk classification** (ADR-0041)
The durable axis that gates which record-only governance review types get a deterministic executor now vs. deferred: **whether the executor rewrites/collapses/spawns/redirects a stable node id** (`concept_id`/`entity_id`/ŌĆ”, with its backlinks, citations, derived edges, and review subjects). **Non-rekeying** executors (`mark_semantic_duplicate`, `hide_content`, and `change_entity_subtype` *only if* it preserves the id) merely add an edge or flip a status/metadata field on existing ids ŌåÆ buildable now, one small slice each. **Rekeying** executors (`merge_entities`, `merge_concepts`, `split_entity`) collapse or spawn ids and must redirect every reference ŌåÆ highest blast radius, **designed in the identity-surgery ADR-0050** (merge + subtype-rekey shipped ([[identity-surgery-merge]] ADR-0050, [[entity-subtype-rekey]] ADR-0051), split shipped ([[entity-split]] ADR-0052)). Reversibility and retrieval-visibility are secondary axes, not the gate (the system already has status/eligibility for visibility; rekeying cuts deeper because it changes what ids *mean*, ADR-0021). **`mark_semantic_duplicate`** is the first executor (ADR-0041): on approve it upserts a single **active, canonical, symmetric `duplicates(min_id, max_id)`** assertion (`asserted_by: human`) between two same-type nodes and re-renders a **body-only** `## Duplicates` backlink section on both pages (projected for any active edge regardless of either node's lifecycle status, like all graph projections; human-navigation only, confers no answer-eligibility) ŌĆö **pure annotation**: no id rewrite, no node-status change, no retrieval suppression, no backlink redirect. Canonical ordering + the wikiŌåögraph projection are validator-enforced (`validate_graph` rejects a reversed `duplicates` row; `validate_projection` gains both-direction `duplicates` coverage). Subject is the canonicalized pair `{node_ids: sorted([a,b])}`; the executor scope-guard-skips a malformed/invalid/self/cross-type/missing pair (never partial-applies); reject is a ledger-only no-op; it is graph-required and previewable via the [[apply-dry-run]] (edges_added + two wiki diffs; apply does not move approved review files, so no `diff.reviews` entry). A re-apply of an already-projected duplicate is a true no-op (applied/normalized/no-op semantics).

**Identity surgery / merge** (ADR-0050, implemented ŌĆö the rekeying class deferred by ADR-0041)
The highest-blast-radius governance class: collapsing/spawning stable node ids. **Model:** **hard re-point + tombstone, FORWARD-ONLY, auditable (not live-reversible) in v1** ŌĆö *a merge rewrites live identity references to the survivor and leaves the absorbed id as a durable tombstone/redirect record*. Soft-redirect (a runtime `merged_into` alias every reader follows) is **rejected** (pervasive read-path complexity + drift; diverges from graph-is-SoT). **v1 ships `merge_entities` + `merge_concepts` for EXACT same node_type only** (`per_`+`per_`, ŌĆ”, `cpt_`+`cpt_`); **prefix-changing `change_entity_subtype` (single-node rekey) shipped as [[entity-subtype-rekey]] (ADR-0051) and `split_entity` shipped as [[entity-split]] (ADR-0052); cross-type merge and any live un-merge remain deferred** (documented, own follow-ups). **Surface taxonomy** (what's a "live identity reference"): graph **edges** (key on id) ŌåÆ **hard re-point BŌåÆA**; **wikilinks** ŌåÆ the **tombstone resolves literal `[[ŌĆ”/B-slug]]`** (no global relink; graph-PROJECTED backlinks update via edge re-point + page re-render); claim **citations** (source-keyed) ŌåÆ **N/A**; **search/nav rows** ŌåÆ auto-reindex; **unresolved review subjects in `pending/`** (status `pending` OR `deferred`) referencing B ŌåÆ **withdrawn via the existing `reviews.withdraw_review_item`** (removes `pending/<id>.json` + an `audit_log/<id>-withdrawn-<hex>.json` entry, `note: superseded_by_merge`; "withdrawn" is an audited operation, NOT a new item status ŌĆö REVIEW_STATUSES stays pending|approved|rejected|deferred; **rejected** items = immutable audit, **approved** items are checked by the approved-unapplied gate below). **Edge re-point invariant:** *the live graph after merge ŌēĪ "replace B with A, then normalize: canonicalize symmetric edges, collapse duplicates, remove self-edges"* ŌĆö re-point is an in-place endpoint UPDATE preserving `edge_id` + provenance (asserted_by/review_id/job_id/evidence); **active edges only** (historical/superseded rows stay as audit). **Collision key = the FULL assertion identity** `(src,dst,edge_type,asserted_by,evidence_source_id,char_start,char_end)` (the `uq_edges_assertion` index) ŌĆö so edges differing only in evidence anchors are DISTINCT and **coexist** (two `mentions(SrcŌåÆA)`+`mentions(SrcŌåÆB)` with different spans ŌåÆ two distinct active `mentions(SrcŌåÆA)`); only a full-identity collision collapses (absorbed row ŌåÆ `status: superseded`, NOT deleted; absorbed edge id in the merge audit). Self-edges (AŌåöA) ŌåÆ `superseded` (not deleted; absorbed rows keep their B endpoint, status-changed only). **Canonicalize only `{contradicts, duplicates}`** (per validate_graph); **`supersedes` is DIRECTED** ŌĆö preserve direction (`supersedes(BŌåÆC)`ŌåÆ`(AŌåÆC)`), never reverse. Re-pointed edges must stay valid under `EDGE_ENDPOINTS` (else the `invalid_repoint_endpoint` pre-write block, below ŌĆö never a skip). **Inactive-collision (uq_edges_assertion is status-AGNOSTIC):** a re-point can hit an inactive A row of the same full identity ŌåÆ **resurrect** the target to active on `proposed`/`superseded` (absorbedŌåÆsuperseded, audit `resurrected_target_collision`), **BLOCK the merge** on `rejected` (`rejected_target_collision` ŌĆö never auto-override a human rejection). **Pre-merge BLOCK gates (never partial-apply; dry pass detects, refuses before any write):** `rejected_target_collision` + `invalid_repoint_endpoint` (a re-point would violate `EDGE_ENDPOINTS` ŌåÆ BLOCK, not skip ŌĆö else an active edge keeps a merged endpoint) + **`approved_unapplied_references_absorbed`** (scan `reviews/approved/` for B-referencing items; block if any projects PENDING_APPLY/UNKNOWN/APPLY_DEFERRED; EFFECTED/NO_EFFECT_REQUIRED don't block). **Withdrawal** covers all `pending/` items (status `pending` OR `deferred`); the **single subject matcher** (used by withdrawal AND the approved gate) is exact node-id equality in structured fields ŌĆö `node_id`/`node_ids[]`/`survivor+absorbed`/**`topic_node_id`** (propose_synthesis keys its subject on the topic)/page-path ŌĆö never prose. Resurrecting a `proposed` target edge **rewrites its `review_id` to the merge review id** (the activation was authorized by the merge, not the old proposal ŌĆö the explicit exception to "re-point preserves provenance"; asserted_by/evidence/job_id stay) and **withdraws the old proposal's pending/deferred review** (`superseded_by_merge`; it's on A, so the B-matcher doesn't reach it); the merge audit records `previous_target_review_id`. **Alias union:** A.title unchanged; `aliases(A) := stable-dedup(A.aliases ++ [B.title] ++ B.aliases)`. Reindex-fail warning pinned `merge_discovery_reindex_not_guaranteed`. **New `merged` lifecycle status** (Ōłē `RETENTION_DEFAULT_STATUSES` ŌåÆ out of nav/graph/answer-eligibility; **not** `deprecated_candidate`, which stays discoverable): absorbed node B stays at its old path as a tombstone preserving the **full required frontmatter schema** (`type`/typed-id/`title`/`confidence`/`review_status`) + `status: merged`, `merged_into: <A>`, `merged_at`/`merge_review_id`, aliases preserved; only the **body** collapses to a short "Merged into [[A]]" note; survivor **A stays active** with B's title+aliases **unioned in** (merge is the explicit place that writes page-authoritative aliases). **Subject = explicit winner** `{survivor_node_id, absorbed_node_id}` + `proposal {to_status: merged}` (unlike the unordered `duplicates` pair); guards skip type-mismatch/self/missing/non-active/already-merged before any write; reject = no-op; **graph-REQUIRED**; reopen only while PENDING (applied = `EFFECTED`, forward-only). **Reconstructable audit** in the existing `reviews/audit_log/` (one structured `<review_id>-merged-<hex>.json` entry: absorbed/survivor ids+paths, re-pointed/collapsed/dropped edge ids with original provenance, re-rendered pages, unioned aliases, withdrawn subjects, review id/actor/time; no separate artifact store) is the basis of the auditable (not-live-reversible) posture ŌĆö a future inverse is a NEW governed op. **Validator invariant:** a `merged` node's `merged_into` points to an active same-type survivor, and **no active edge may have a `merged` endpoint**. Reindex-failure non-clean (`merge_discovery_reindex_not_guaranteed`), previewable via the [[apply-dry-run]].

**Entity subtype rekey / `change_entity_subtype`** (ADR-0051, implemented ŌĆö the second rekeying class, deferred by ADR-0041/0050)
The **smallest** identity-surgery executor: a **single-node 1:1 relabel** that changes one entity-family node's
subtype, and therefore its **id-prefix and page directory** (`ent_`/`per_`/`org_`/`prj_` ŌåÆ `Entities`/`People`/
`Organizations`/`Projects`, ADR-0021) ŌĆö a **rekey**, not the id-preserving change ADR-0041 classed non-rekeying,
and **not** a merge (no 2ŌåÆ1 collapse, no source-set union). Wires an executor onto the registered-but-record-only
`change_entity_subtype` review type, **reusing ADR-0050 machinery** (`graph.repoint_edge`, dry-plan-then-apply,
`invalid_repoint_endpoint` + approved-unapplied block gates, tombstone render seam, Source re-render fan-out,
`reviews.withdraw_review_item`, `reviews/audit_log/`, the ADR-0040 dry-run). **Scope v1:**
entity-family **only** (`entity Ōåö person Ōåö organization Ōåö project`, any direction); **concept and any
conceptŌåöentity move excluded** (a cross-family *type* change ŌĆö a future `change_node_type`); `split_entity`,
cross-type merge, **collision-as-merge**, and a live **un-rekey** deferred (forward-only, auditable not
live-reversible). **Subject** `{node_id, to_type}` (the proposed identity change is the review unit; mirrors
merge's `{survivor,absorbed}` ŌĆö a bare `{node_id}` would, via `create_review_item` idempotency across pending/
approved/**rejected**, permanently lock a node out of *any* future retype once one target is rejected) +
**proposal** `{to_type}` (**must equal `subject.to_type`** else `to_type_mismatch`); `_subject_references`
still keys on `subject.node_id` so withdrawal/approved-gate work + **applying one retype withdraws competing
pending/deferred retypes of the same node**; **no proposal-update/supersede semantics** (re-proposing the same
pair is an idempotent no-op). **Old id must be a CANONICAL shape** (`noncanonical_node_id` skip if it doesn't
fullmatch `^(cpt|ent|per|org|prj)_[0-9a-f]{16}$` ŌĆö tighter than merge's `_is_safe_id`, mirrors `CLAIM_ID_RE`/
`_SOURCE_ID` + the `f5ba86a` hardening; a well-formed `cpt_` concept id passes shape and is then caught by the
family guard as `out_of_scope`, the honest reason) before deriving the new id. **New id = prefix substitution on the FROZEN hash, never
re-hashing title/name** ŌĆö `new_id = prefix(to_type) + "_" + old_id.split("_",1)[1]` (`ent_abcŌĆ” ŌåÆ org_abcŌĆ”`):
forced both by the validator invariant and by ADR-0021's frozen-id model (a renamed node's title no longer
hashes to its id), and **tamper-proof** (no untrusted name feeds the id). The new node copies title/aliases/
confidence from the old **page** and **derives source links from the re-pointed active `mentions`** (`graph.
sources_for_node` ŌĆö concept/entity pages don't own `source_ids` in frontmatter); only id-prefix + directory
change (name-derived slug unchanged). **Crux A ŌĆö virgin-target-only, THREE apply-time block gates** (dry-plan,
before any write; surfaced in the dry-run since colliding state can appear between approval and apply):
**`target_subtype_id_exists`** (status-agnostic `graph.get_node(new_id) is not None` ŌĆö active/candidate/
deprecated/hidden/archived/merged/rekeyed/deleted), **`target_subtype_page_exists`** (target page on disk ŌĆö
NOT redundant: `wiki/`/nodes-table can **drift**, an orphan page would otherwise be silently overwritten), and
**`target_assertion_exists`** (`find_assertion` on any re-pointed full identity ŌĆö a drifted/tampered graph can
carry a dangling row that `repoint_edge` would hit *after* minting). Never a silent merge or tombstone
resurrection; rekey **blocks** on a pre-existing target assertion (never collapse/resurrect ŌĆö that's the
identity-collapse it forbids). **Crux B ŌĆö new `rekeyed` tombstone status** (Ōłł NODE_STATUSES + validate_wiki + retention.yaml,
**Ōłē RETENTION_DEFAULT_STATUSES** ŌåÆ drops from default nav/graph/answer-eligibility for free) + **`rekeyed_to`**
pointer (`rekeyed_at`/`rekey_review_id`, full frontmatter schema, body collapses to a redirect note that keeps
the standard `> [!summary]` callout ŌĆö `> [!summary] Retyped <type>` + "Retyped into [[ŌĆ”]]", mirroring the
`merged` tombstone);
**`merged` is NOT generalized** (it stays strict same-type ŌĆö `merged` = absorbed into a different node, lossy;
`rekeyed` = same logical node relabeled, 1:1). `validate_graph`: no active edge may touch a `rekeyed` endpoint.
`validate_projection`: `rekeyed_to` ŌåÆ **active-or-candidate** (the mint preserves the old status),
**different-type**, same-family, non-self, indexed ŌĆö **and same name-hash, prefix-only delta** (a sharper
check than merge's same-type rule can express). **Crux D ŌĆö retypable iff
`status Ōłł {active, candidate}`** (the new node **preserves** the old status; **diverges from merge's active-only**
because a 1:1 relabel carries the same mentions/sources ŌåÆ no promotion-accounting hazard, and the dominant
trigger ŌĆö correcting a classifier-default generic `ent_` candidate ŌĆö happens *before* promotion); all other
statuses skip `node_not_retypable`; a pending `promote_candidate_node` for the old id is withdrawn. **Apply
(graph-REQUIRED; re-point BEFORE render so the new page's source links populate; simpler collision policy than
merge ŌĆö in a consistent graph the virgin target has no edges so the `find_assertion` check never fires, and on
drift it BLOCKs rather than collapse/resurrect):** dry-plan + 3 gates ŌåÆ **upsert bare new node** ŌåÆ `repoint_edge`
oldŌåÆnew (provenance preserved, re-canonicalize `{contradicts,duplicates}`) ŌåÆ **render new page** (`sources_for_
node(new_id)` now populated) ŌåÆ tombstone old ŌåÆ fan-out re-render (**Source `mentions` pages only** ŌĆö an
active `duplicates` edge BLOCKS the rekey via `invalid_repoint_endpoint` [SAME_TYPE_EDGES broken by the type
change], so no partner ever survives to re-render; `related_to` is unprojected (ADR-0031) ŌåÆ audited, no
re-render) ŌåÆ withdraw old-id subjects (incl. competing retypes) ŌåÆ audit. `noop_same_type` is a
**typed no-op** (mutates/blocks nothing); `invalid_repoint_endpoint` blocks a `duplicates`-to-same-old-type
partner (resolve the duplicate first); **no global literal-link rewrite** (literal `[[Entities/slug]]` resolves to
the tombstone redirect). **Projector** `_effect_rekey`: `EFFECTED` (old `rekeyed`+`rekeyed_to`+new present+approved)
/ `PENDING_APPLY` (old still live) / **`UNKNOWN partial_rekey_state`** (half-applied ŌåÆ not reopenable);
reindex-failure ŌåÆ non-clean `rekey_discovery_reindex_not_guaranteed`. Previewable via the [[apply-dry-run]].

**Entity split / `split_entity`** (ADR-0052, implemented ŌĆö the last & highest-blast-radius identity-surgery op)
The **inverse of merge**: one entity-family node's evidence is divided into a surviving **primary** and a
freshly-minted **spin-off**. Wires an executor onto the registered record-only `split_entity` type, reusing
ADR-0050/0051 machinery (`graph.repoint_edge`/`find_assertion`, dry-plan-then-apply, virgin-target gates,
`reviews/audit_log/`, ADR-0040 dry-run). **Crux 1 ŌĆö Option A (original survives), NOT tombstone-and-mint-two:**
the original **A keeps its id AND name unchanged** (the primary); the other half **B is minted fresh**
(id from its new name); only the human-assigned partition of A's evidence re-points to B. Rejected the
tombstone-two model because a split tombstone has **two** redirect targets (ambiguous ŌĆö merge/rekey redirect
to one); Option A has **no tombstone, no redirect, no new lifecycle status, no rename dependency** (there is
NO `rename_node` executor ŌĆö ADR-0017 rename is design-locked-but-unimplemented, which bounds this), re-points
the **least**, and its rollback reuses `merge_entities(spin-off ŌåÆ primary)` **once the spin-off is `active`**
(merge is active-only; B is born candidate). **v1 boundary: the primary
keeps the original name** (both-halves-renamed = split + a future rename, composed; N-way split, subtype-
differing spin-off, moving non-`mentions` edges, live un-split, and a `split_from` graph edge are deferred).
**Crux 2 ŌĆö partition contract:** subject `{node_id, spinoff_node_id}` (both ids, review_id distinct per pair);
proposal `{spinoff_name, spinoff_sources, spinoff_aliases}`; executor recomputes `B = node_id(A.node_type,
spinoff_name)` and **rejects a mismatch** (compute-and-verify). **`spinoff_sources` is the required partition
authority** ŌĆö the mentions that **MOVE** `sourceŌåÆA ŌćÆ sourceŌåÆB` (true partition, **no copy**; a source about
both meanings stays on the primary ŌĆö v1 limit); must be a **non-empty PROPER subset** of A's active mention
sources (both halves keep Ōēź1 ŌĆö all-moved is a rename, none a no-op); each listed source must be a canonical
`src_<16 hex>` (typed `noncanonical_source_id`) that actively mentions A. `spinoff_aliases` optional/advisory
but validated (`ŌŖå A.aliases`; removed from A, added to B); **`spinoff_name` is auto-moved off A** (B owns it as
its title ŌĆö A's final aliases exclude `spinoff_aliases` AND `normalize(spinoff_name)`, else A re-conflates B).
`spinoff_sources`/`spinoff_aliases` are **stable-deduped sets** (first-seen order kept; a duplicate is
normalized, not an operator error).
**Non-`mentions` edges stay on the primary** (rare `related_to`/`duplicates` are A's continuation; moving them
is a follow-up). B keeps A's node_type. **Crux 3 ŌĆö lifecycle + provenance:** the spin-off B is **ALWAYS born
`candidate`, never inherited-active** (split DIVIDES evidence, so promotion must be re-earned; `promote_
candidates` only scans candidates, so an inherited-active B with <2 independent sources would be a fabricated,
never-rechecked promotion ŌĆö the key divergence from rekey's 1:1 status-preservation). The **primary keeps its
status** (no auto-demotion if left with one source ŌĆö documented limit, matches active-page preservation).
Provenance = durable `reviews/audit_log/<rid>-split-<hex>.json` (authority, since `wiki/` is regenerable) +
**page-preserved** `split_from`/`split_review_id` frontmatter on B (advisory lineage, NOT a validator-enforced
redirect ŌĆö B is live) + **no graph edge** (a new edge_type for zero query use is disproportionate). **Crux 4 ŌĆö
apply (new `app/workers/splits.py::apply_splits`, GRAPH-REQUIRED, after `apply_rekeys`; never partial):**
subject/partition guards + **three virgin-target block gates on B** (`target_spinoff_id_exists`/
`target_spinoff_page_exists`/`target_spinoff_assertion_exists`) + **`spinoff_promote_slot_taken`** (a terminal
approved/rejected promote record for B blocks; pending is reused ŌĆö a virgin node ŌēĀ virgin ledger slot) +
**`approved_unapplied_references_primary`** (an approved-but-unapplied item referencing A blocks, same
effect-status gate as merge/rekey ŌĆö split shifts A's evidence and `promote_candidates` promotes A on
`pre_approved` alone) ŌåÆ upsert bare B(candidate) ŌåÆ repoint moved
mentions ŌåÆ render B (+`split_from`) ŌåÆ re-render A (aliases minus spinoff_aliases, status unchanged) ŌåÆ **file
B's `promote_candidate_node`** (enters the promotion ledger) ŌåÆ fan-out re-render the **moved sources' Source
pages** (the only fan-out) ŌåÆ audit. **No pending-review withdrawal** (nothing is retired ŌĆö the clean contrast
with merge/rekey). **Projector `_effect_split` ŌĆö `EFFECTED` iff:** B node exists (candidate|active) + B page
`split_from==A`+`split_review_id==rid` + each spinoff_source now mentions B and not A + A retains Ōēź1 mention +
**B's promotion is accounted for** (pending/approved/rejected promote item exists ŌĆö a terminal *rejected*
promote is a filled ledger slot, deliberate human accounting, not a partial ŌĆö or B already active); a half-mint
(e.g. mentions moved + B rendered but promote item not yet filed) ŌåÆ **`UNKNOWN partial_split_state`** (not
reopenable, mirrors rekey half-mint safety). Reindex-failure ŌåÆ non-clean `split_discovery_reindex_not_guaranteed`.
Forward-only/auditable; inverse = `merge_entities(spin-off ŌåÆ primary)`. Previewable via the [[apply-dry-run]].

**Hidden status / `hide_content`** (ADR-0043)
The second non-rekeying governance executor: an approved **`hide_content`** review suppresses a **source** from default retrieval + navigation via a new reversible **`hidden`** lifecycle status ŌĆö **no id rewrite, no deletion; raw + page stay on disk**. It is a near-clone of [[retention-lifecycle]]'s `archive_source`/`apply_archive_sources`: `active ŌåÆ hidden` only (manifest `status` is authority ŌåÆ Source-page mirror ŌåÆ best-effort graph source-node mirror ŌåÆ keyword/nav reindex), with `hidden` added to the status vocabulary but **kept out of `RETENTION_DEFAULT_STATUSES`**, so it is excluded from `/query` evidence + `/search` (incl. the navigation + graph channels) **for free** ŌĆö like `archive_candidate` (the levers for a *source* are the evidence-retrieval source-status filter + navigation; Source pages are already `answer_eligible: false` regardless of status, so `answer_eligible` is a lever only for the deferred semantic-page slice). `hidden` is **distinct from `archive_candidate`** (governance suppression vs retention staleness; different review types/audit, same visibility mechanism, separately filterable). What "hidden" means is **retrieval + navigation only, NOT graph traversal**: graph APIs still reach a hidden node with its status visible (provenance/reversibility), a backlink may still render on another page (documented v1 limit), and `wiki/index.md` still lists it annotated `hidden`. Scope: **sources only** in v1 (`subject.source_id`; guards mirror `archive_source` ŌĆö `invalid_source_id`/`source_missing`/`unexpected_to_status`/already-not-active no-op; reject = no-op; executor-backed but **not graph-required**); semantic-node-page hiding and an **unhide** (`hidden ŌåÆ active`) executor are deferred. Previewable via the [[apply-dry-run]] (manifest `active ŌåÆ hidden` diff + Source-page diff).

**Apply Dry-Run / Mutation Preview** (ADR-0040)
The read-only preview of what `POST /reviews/apply` *would* change, served by `POST /reviews/apply/dry-run`. It is **apply-on-a-copy, not an independent projection**: it builds a **fully self-contained** throwaway sandbox root with **no live path reachable from it** ŌĆö copying the writable domains (`db/` minus `llm_cache`, `reviews/`, `wiki/`, `raw/manifests/`) and the read-only inputs the orchestration touches (`scripts/`, `templates/`, `policies/`, `normalized/`, and the **manifest-referenced** raw bytes so the full validator suite incl. `validate_raw_integrity` runs faithfully) ŌĆö runs the **same** extracted `run_apply(settings)` orchestration (the 5 executors ŌåÆ `wiki/index.md` rebuild ŌåÆ keyword reindex ŌåÆ validator suite) against the sandbox, then diffs sandbox-vs-live and discards the sandbox ŌĆö so **executor behavior cannot drift**. Live raw bytes are read-copied into the sandbox but **never modified**; `indexes/`/vector/keyword/`llm_cache` are not copied (regenerated in-sandbox if needed, excluded from the diff). The graph-availability gate lives **inside `run_apply`** so live and dry-run refuse identically (live ŌåÆ HTTP 503; dry-run ŌåÆ a structured `status:"blocked"` / `graph_unavailable` preview). The result is a domain-grouped semantic **diff** (`graph` edge/node-status deltas with stable ids, `wiki` path-scoped unified diffs, `reviews` workflow moves, `manifests` field-level status) plus an `items[]` provenance list (review_id/type/targets ŌåÆ effects), a `not_appliable[]` list (record-only/unsupported types, with reasons ŌĆö never a fabricated diff), `validators` (would-pass signal; a sandbox validator failure makes it a **failed preview** and the UI withholds Apply), and `warnings[]`. Distinct from the per-type `ReviewPreview` projector (ADR-0035 A1), a single-item UI hint that is *not* the executor path. Dry-run passing is **advisory, not a lock** ŌĆö live apply still validates after mutation. Sandbox executor failure is a structured failed dry-run with diagnostics, not a 500; a no-op yields an empty diff.

**Synthesis visibility / hide_synthesis + unhide_synthesis** (ADR-0049)
The hide/unhide pattern extended to the **last** generated surface ŌĆö **synthesis pages** (the `synthesis.py` / `render_synthesis_page` seam) ŌĆö completing the visibility family (sources, concept/entity, claims, synthesis all symmetric). Visibility-only, **no edge deletion**; `hide_synthesis` (activeŌåÆhidden) + `unhide_synthesis` land **together**; new `apply_hidden_syntheses`/`apply_unhidden_syntheses` executors in `synthesis.py`, called from `run_apply`'s graph block **alongside `apply_resolved_syntheses`** (which already wires `synthesis_dir`/`enrichment_dir`), **GRAPH-REQUIRED**. **Active-only Ōćä active:** hide `active ŌåÆ hidden`+`review_status: approved` (a `candidate`/`deprecated_candidate` is a typed `synthesis_not_active` skip ŌĆö a candidate is already non-discoverable, `candidate Ōłē RETENTION_DEFAULT_STATUSES`, and is disposed of by rejecting its `propose_synthesis`); unhide `hidden ŌåÆ active`+**`review_status: approved`** ŌĆö restoring the **synthesis** active convention (a promoted synthesis is `approved`, *not* the `none` concept/claim unhide restores). Unhide is **blind active** (key-free, no eligibility recompute); a topic that lost eligibility while hidden is re-tombstoned by the **next generate pass**, self-correcting. **Synthesis-specific crux ŌĆö preservation (hide wins):** `hidden` is a human override; `generate_syntheses` must never clobber it, so a `hidden` guard is added at its **three** clobber sites ŌĆö the **retraction loop** (skip hidden like `deprecated_candidate`; retract-through-hidden is **rejected** ŌĆö `deprecated_candidate Ōłł RETENTION_DEFAULT_STATUSES` would *re-expose* hidden material), **`apply_resolved_syntheses`** (a lingering proposal can't flip a hidden node), and the **normal-regen gate** (`if status=='hidden': continue` ŌĆö **no LLM call**, no node reset). Each guard keys on `hidden` from the **authoritative page** OR the graph mirror (review fix: a page-hidden/graph-active partial state is also preserved), **skip-only** (no mirror repair ŌĆö that's the executor/validator's job). The hide/unhide executors are the **only** paths in/out of `hidden`; source drift is reconciled after unhide. **No cross-page fan-out** (unlike the claim crux): nothing renders `[[Synthesis/ŌĆ”]]` (the conceptŌåÆsynthesis `related_to` is unprojected, ADR-0031), so hiding touches **only** the synthesis page + node + retrieval surfaces. **Render:** `render_synthesis_page` gains a `hidden` branch ŌĆö "Synthesis hidden ŌĆö suppressed from default discovery" banner, **Supporting Evidence + Disagreements sections kept** (the page is the durable inspection record; hide suppresses discovery, not the record). **Effect:** drops from default `/search` nav + graph channel + `answer_eligible` (synthesis IS answer-eligible) ŌĆö **not** chunk evidence / `/query` citations (source-chunk-keyed, ADR-0034); raw `/graph/*` still shows it + its `derived_from`/`related_to` edges, `status: hidden`. Render-source = the synthesis **artifact** (missing ŌåÆ typed `synthesis_artifact_missing` skip), via `_render_page`. **Untrusted-page binding (review hardening):** `topic_node` (the artifact key) is read from the untrusted page, so the executor + `rerender_synthesis_page` require `synthesis_id(topic_node) == nid` (else `synthesis_topic_mismatch`) + artifact `node_id == nid` (else `synthesis_artifact_mismatch`) ŌĆö a tampered page can't re-render this synthesis with another topic's prose. **Hidden-claim fan-out (review fix, extends ADR-0048):** a synthesis's Supporting Evidence + `derived_from` frontmatter are default-discovery surfaces, so `_render_page` **omits hidden claims** (active AND hidden synthesis pages filter; the edge stays active in the graph), `validate_projection` matches links to active **non-hidden** claims, and **claim hide/unhide re-renders citing syntheses** (the ADR-0048 claim executor returns `affected_syntheses` ŌåÆ `run_apply` ŌåÆ `synthesis.rerender_synthesis_page`) ŌĆö closing a route where an active synthesis page linked a hidden claim. The same review fixed a **pre-existing ADR-0048 gap**: `validate_projection` now expects active **non-hidden** claim endpoints on all three rendered surfaces (Source-page Claims, Claim-page Contradicting-Claims, Synthesis Supporting Evidence), matching the renderers (a real vault with a hidden claim + Source/partner pages previously validate-failed after a correct apply). **Evidence-suppression (`evidence_hidden`, ADR-0049 decision 10):** synthesis PROSE is LLM-derived from its claims, so filtering only the *link* still leaves hidden-claim content discoverable through the `## Synthesis` body. A new node status **`evidence_hidden`** (Ōłł NODE_STATUSES + validate_wiki + retention.yaml, Ōłē RETENTION_DEFAULT_STATUSES ŌåÆ out of nav/graph/answer_eligible) auto-suppresses an **active** synthesis materially derived from a hidden claim ŌĆö **distinct from operator `hidden`** (clean restoration rules: operator hide reversed only by unhide_synthesis; `evidence_hidden` reversed by claim unhide when ALL supporting claims visible). Precedence (recomputed by the claim fan-out `rerender_synthesis_page`): operator `hidden` **wins** > `evidence_hidden` (any active derived_from claim hidden) > evidence-derived; only `active Ōćä evidence_hidden` (candidates/tombstones aren't default-discoverable). Operator unhide with hidden evidence lands on `evidence_hidden`; the generate pass preserves it (`_PRESERVED_GENERATE_STATUSES`, skip-only); edges/prose/artifact intact (raw inspection); banner "Synthesis suppressed ŌĆö supporting evidence hidden". **Apply order:** `run_apply` runs claim hide/unhide ŌåÆ explicit `hide_synthesis`/`unhide_synthesis` ŌåÆ claimŌåÆsynthesis fan-out (final reconciliation), so operator `hidden` wins in a mixed batch; an unreconcilable fan-out (missing artifact ŌåÆ `rerender_synthesis_page` returns None) is **non-clean** (`validation_failed` + `synthesis_evidence_suppression_not_guaranteed`). `run_apply` audits `synthesis_evidence: {suppressed, restored, unreconciled}`. **Repair-then-rerun:** the claim executor recomputes `affected_syntheses` on EVERY apply (incl. fully-effected claims; only `claim_not_active` excluded) so a once-failed fan-out is retried after the page/artifact is repaired; `_render_page`/`rerender_synthesis_page` are **change-detecting** (byte-stable page rewritten + node-status upserted only when they differ) so a steady-state apply is a true no-op (no churn/reindex). The fan-out computes its precedence target from the **authoritative PAGE status** (not the stale graph mirror ŌĆö a page-hidden/graph-active partial isn't downgraded to evidence_hidden; the mirror is repaired to match); a graph-mirror-only repair still counts (`synthesis_fanout_work`) so reindex runs, and is non-clean if reindex fails. **Known follow-up (non-blocking):** the retry covers the synthesis fan-out but not the ADR-0048 Source-page/contradiction-partner re-render for fully-effected claims (deferred ŌĆö those have no artifact-missing failure mode and generate_wiki isn't change-detecting, so collecting them for effected claims would churn). v1 limit: hide_synthesis stays active-only (can't directly operator-hide an evidence_hidden synthesis). **Projector** keys on hidden-ness (hide `EFFECTED`=fully hidden+`approved`, unhide `EFFECTED`=not hidden; partial = `UNKNOWN partial_hide_state`/`partial_unhide_state`, not reopenable; `synthesis_not_active` warning). **Reindex-failure** ŌåÆ non-clean + `synthesis_hide_discovery_suppression_not_guaranteed`/`synthesis_unhide_discovery_restoration_not_guaranteed` (per-surface naming; graph-only completion still reindexes; `index.md` keeps it listed annotated `hidden`).

**Claim visibility / hide_claim + unhide_claim** (ADR-0048)
The ADR-0046/0047 hide/unhide pattern extended to **claim pages** (the distinct `recompose_claim` seam) ŌĆö visibility-only, **no edge deletion**; contradiction detection is **status-filtered, not edge-destructive**. `hide_claim` (activeŌåÆhidden) + `unhide_claim` land **together**; separate `apply_hidden_claims`/`apply_unhidden_claims` executors, GRAPH-REQUIRED. **Status mechanism (claim-specific):** claim status is **evidence-derived**, so `recompose_claim` gains a **`hidden` governance status** with precedence over evidence + **preserved across re-render** (like `deprecated_candidate` ŌĆö a hidden claim never silently re-derives active). Hide ŌåÆ `hidden`+`approved` (active-only); **unhide CLEARS the override and RE-DERIVES** ŌåÆ `active`+`none` if cites remain, or tombstone `deprecated_candidate` if evidence was lost (never active-with-zero-citations). **Crux ŌĆö rendered contradiction backlink section:** a hidden claim X is **omitted by default** from a partner claim Y's "Contradicting Claims" section (filter partner by visible status; there is no supersedes *partner* section ŌĆö "superseded" is only a label on the loser's own page), and hiding/unhiding X **re-renders the affected partner pages** (the `contradiction_affected` fan-out); the **edges stay active** (raw `/graph/*` still shows XŌåöY, `status: hidden`). **Effect:** drops from default `/search` **navigation + graph channel + answer-eligibility** (claims ARE answer-eligible) ŌĆö **not** chunk evidence (claims aren't `/search` evidence or `/query` citations; both are source-chunk-keyed, ADR-0034). **Detection:** no edge surgery, but a hidden claim is non-active so it's excluded from **future** contradiction candidate generation by the existing `active_node_ids_of_type("claim")` gate (existing edges preserved; unhide re-admits it **only if it re-derives active** ŌĆö a re-derived tombstone stays excluded). Source-page Claims sections + partner contradiction sections also omit hidden claims and re-render on hide/unhide. **Projector keys on hidden-ness** (unhidden state may be active OR tombstone): hide `EFFECTED`=fully hidden, unhide `EFFECTED`=not hidden; partial = `UNKNOWN partial_hide_state`/`partial_unhide_state` (not reopenable). Reindex-failure ŌåÆ non-clean + `claim_hide_retrieval_suppression_not_guaranteed`/`claim_unhide_discovery_restoration_not_guaranteed`.

**Unhide / unhide_content + unhide_semantic_page** (ADR-0047)
The governed **inverse** of the hides: a human-approved review item reverses an **already-live** hide (`hidden ŌåÆ active`) via apply ŌĆö the `EFFECTED`-state effect-reversal ADR-0045 [[reopen-re-decide]] deferred. **Not a reopen:** reopen rewinds a *not-yet-applied decision* (PENDING_APPLY) back to pending; unhide is a *new governance action* against a *live status*. **Separate types** (`unhide_content` source / `unhide_semantic_page` semantic ŌĆö intent-encoding, not bidirectional hide types); `proposal.to_status: active` is a guard. **v1 scope:** sources + concept/entity-family (claim/synthesis unhide await their hide). **Authority:** source = manifest (`set_status(active)`, reuses `_apply_source_status_transition` generalized with a `from_status` param ŌĆö hiddenŌåÆactive), best-effort graph mirror, **not graph-required**; semantic = page+graph via `recompose_semantic_node_page(status="active", review_status="none")` ŌĆö restores the **clean default active state** (uniform with never-hidden pages; governance trail in the ledger/audit), **graph-required**. **Effect:** restores default `/search` nav + graph-channel discovery + `answer_eligible`; reindex-failure ŌåÆ non-clean (`validation_failed` + **`unhide_discovery_restoration_not_guaranteed`** ŌĆö inverse risk: page active on disk but a stale index still hides it from discovery); `index.md` drops the `hidden` annotation. **Projector/reopen-safety (inverse of [[semantic-page-hiding]]'s `partial_hide_state`):** `EFFECTED` = fully active; `PENDING_APPLY` = still fully hidden (reopenable ŌĆö no live unhide effect); **semantic partial** (page XOR graph active) ŌåÆ `UNKNOWN partial_unhide_state` (not reopenable). **Source has no partial state** ŌĆö the manifest is the single authority (page is a projection), so `_effect_unhide_content` is `EFFECTED` iff the manifest is active, with a stale page as a `page_manifest_drift` warning (mirrors source [[hide-content]]). **Hidden-only guard:** a non-hidden subject ŌåÆ `source_not_hidden`/`node_not_hidden` skip; reject ŌåÆ no-op.

**Semantic-page hiding / hide_semantic_page** (ADR-0046)
The ADR-0043 source-hide model extended to generated **semantic node pages**: an approved `hide_semantic_page` flips a concept/entity-family node **`active ŌåÆ hidden`**, reusing the `deprecate_wiki_page` executor's render seam + guards (`recompose_semantic_node_page(status="hidden")`, subject `{page, node_id}`, canonical-page guard, scope-dir gating). **v1 scope:** concept/entity/person/organization/project (claim + synthesis are fast-follows on their own seams). **Authority:** page frontmatter (authoritative, rendered at `status: hidden` + `review_status: approved`) + graph node (mirror) written together ŌĆö so it is **graph-REQUIRED** (graph absent / node missing ŌåÆ block + 503; no page-only partial hide), unlike the manifest-authoritative source [[hide-content]]. Stricter reindex posture (mirrors ADR-0043): an applied hide + failed keyword reindex ŌåÆ apply non-clean (`validation_failed` + `semantic_hide_retrieval_suppression_not_guaranteed`); a failed `index.md` *rebuild* stays warning-only. **Effect** (one status flip cascading through existing filters): excluded from default **`/search` navigation** + the **`/search` graph channel** (as adjacent/waypoint) by the `RETENTION_DEFAULT_STATUSES` node-status filter, and its **`answer_eligible` metadata flips false** (needs `status==active`; surfaced on nav rows + `/graph/*` node metadata). `/query` cites **source-chunk evidence** keyed by `source_id` (ADR-0034) ŌĆö semantic pages are never `/query` citations regardless of status, so hiding changes default `/search` discovery, not `/query`. **Preserved**: the graph node + all its **edges** (graph is SoT ŌĆö no edge deletion / backlink rewrite) and raw `/graph/*` inspection (still returns it with `status: hidden`). Reuses the `hidden` status (no new status); **active-only, hide-only v1** (non-active ŌåÆ skip/no-op; reject ŌåÆ no-op); **unhide (`hidden ŌåÆ active`) deferred** (a new executor direction, distinct from [[reopen-re-decide]] which reverts a *decision* not a *status*). Reopen + [[apply-dry-run]] integrate free via the `_effect_hide_semantic` projector: `EFFECTED` iff page `hidden` + `review_status: approved` + graph node `hidden`; a **partial live hide** (page XOR graph hidden, or both hidden but review pending) ŌåÆ **UNKNOWN `partial_hide_state`** (NOT reopenable ŌĆö else [[reopen-re-decide]] would orphan the live hidden page/node); `PENDING_APPLY` only when neither is hidden.

**Reopen / re-decide** (ADR-0045)
A controlled, audited exception to ADR-0035's terminal-decision immutability: a human can **reopen** a terminal review item (move `approved/` or `rejected/` back to `pending/` to be re-decided) **only when its decision's effects are not yet live** ŌĆö fixing a misclicked [[supersede-via-ui]] `winner` or a wrong approve/reject **without** building an applied-effect undo. The safety oracle is the **existing A2 projector `effect_status`** (no new applied-flag): reopen is allowed only for the statuses that *prove* no live effect ŌĆö `PENDING_APPLY` (effect verified absent) and `NO_EFFECT_REQUIRED` (owes no world change) ŌĆö and refused (**409**) for `EFFECTED` (live effect ŌĆö out-of-scope applied-undo), `UNKNOWN` (can't confirm ŌĆö repair the read model first), `INVALID_SUBJECT` (tampered), and **`APPLY_DEFERRED`** (`manual_effect_unknown` ŌĆö a no-executor record-only type means "not applied *by this system*," not "no effect": manual-effect types like `delete_raw_file`/`purge_response_cache` run by hand outside `/reviews/apply`, ADR-0036, and the projector can't detect that; a future ADR may allowlist truly-inert types). Surfaced as a dedicated **`POST /reviews/{id}/reopen`** (graph-aware, unlike the graph-free approve) + a `reviews.reopen_review_item` primitive that clears `decided_*`/`decision_note`/`winner` and appends `audit_log/<id>-reopened-<seq>.json` with `{reason, prior_decision, prior_status, prior_winner?}`; a **non-empty `reason` is required** (400 if blank). The `/ui/reviews/{id}` terminal panel shows a **Reopen** button + reason input only when reopenable. A reopened item is back in `pending/`, so apply + the [[apply-dry-run]] naturally exclude it.

**Supersede-via-UI / contradiction winner** (ADR-0044)
The review-UI affordance for resolving a `resolve_contradiction` by picking the **winning** claim (the loser is superseded). It records the choice as an **optional `winner` field on the *approve* decision** ŌĆö a sub-outcome of an approved item, not a new verb or status: `approveŌłģ` = **acknowledge** (the `contradicts` edge flips active, both claims stand), `approve{winnerŌłł{claim_a,claim_b}}` = **supersede** (the existing `apply_resolved_contradictions` writes an active `supersedes` edge winnerŌåÆloser + deprecates the loser to `deprecated_candidate`, `contradicts` stays active for the record), `reject` unchanged. The executor is **reused verbatim** (it already keys off `item["winner"]`); the only new code is recording + validating the winner and the UI. **Decision-time guards** (executor unchanged, dry-run is the backstop): type is `resolve_contradiction`, item pending/deferred, `winner` Ōłł the canonical subject pair (else 400 ŌĆö never a silent acknowledge), `winner` on any other type or on `reject`/`defer` ŌåÆ 400, and **both Claim pages exist with frontmatter `status: active`** (the page frontmatter is the node-status authority, ADR-0022/0030 ŌĆö so the decision endpoint is **graph-free / never 503s**; graph drift is the dry-run/apply's job), else 409. The recorded `winner` is persisted onto the approved item **and into the terminal `audit_log` entry** (conditionally, when present). The HTML detail page renders the two `proposal.sides` + atomic buttons `[Acknowledge][Supersede: A wins][Supersede: B wins][Reject][Defer]`; `/ui/reviews/{id}/decide` translates `supersede_a/b ŌåÆ approve{winner}` (no JS, loopback-only). The winner is **terminal once approved** (ADR-0035 immutability) ŌĆö a misclick is a documented v1 limit (un-approve/re-decide deferred); previewable via the [[apply-dry-run]] (`supersedes` edge add + loser `activeŌåÆdeprecated_candidate` + Claim-page diffs).

**Embedding Adapter**
The provider-agnostic seam that turns text into vectors behind the small `Embedder` surface (`embed(texts) -> vectors` + `dimension`) the indexer and query path depend on, mirroring the [[llm-adapter]] (ADR-0025). `client_from_settings` branches on `EMBEDDING_PROVIDER`: the **default GPU backend is the in-process [[in-process-flagembedding-backend]]** (`flagembedding_bge_m3`, ADR-0053); the HTTP `EmbeddingClient` (`local_http`/`cloud_*`) ŌĆö which speaks the OpenAI-compatible `/embeddings` wire to a **local** server the operator runs (TEI / vLLM / Ollama) ŌĆö is retained as a **CPU fallback** (ADR-0033). Both own **dimension validation**; the HTTP path additionally owns timeout/bounded-retry. A cloud provider is reachable through the HTTP seam only as an explicit, security-gated opt-in (`embedding_api_key`), because sending normalized chunks to a cloud API exports source text across the local-first trust boundary (ADR-0026, ADR-0033).

**In-process FlagEmbedding backend** (ADR-0053)
The default GPU embedding backend: `BgeM3FlagEmbeddingProvider` runs **BAAI/bge-m3** in-process via `FlagEmbedding.BGEM3FlagModel(use_fp16=True, device="cuda")`, selected by `EMBEDDING_PROVIDER=flagembedding_bge_m3`. It reverses ADR-0033 decision 1 (HTTP-only, no in-process Torch) because **TEI/Candle falls back to CPU on the RTX 5090**; PyTorch+FlagEmbedding runs BGE-M3 on CUDA. **Dense-only v1** ŌĆö `encode(..., return_dense=True, return_sparse=False, return_colbert_vecs=False)`; only `dense_vecs` (dim **1024**) is used; **sparse + ColBERT are deferred** for later hybrid/rerank. Torch/FlagEmbedding are imported **lazily inside methods** and are **installed out-of-lock** (a local accelerator overlay, **not** a locked `pyproject`/`uv.lock` group ŌĆö a cu128 Torch build can't be locked from PyPI), so the core install + the default (fake-embedder, key-free) **CI stay Torch-free** ŌĆö the light path never imports Torch. The model is a **process-level singleton** (loaded once, ~2 GB). Its staleness identity is `resolve_model_ref(settings)` = `flagembedding_bge_m3:<EMBEDDING_MODEL_ID>:<fp16|fp32>` ŌĆö **precision folded in** (fp16-vs-fp32 is a systematic vector difference; device/batch_size/max_length excluded; model id floats ŌåÆ operators force-reindex on an upstream revision change), distinct from the TEI ref ŌåÆ switching backends forces a `--force` rebuild, computed **torch-free** so the offline validator stays offline. Config: `EMBEDDING_MODEL_ID`/`EMBEDDING_DEVICE`/`EMBEDDING_USE_FP16`/`EMBEDDING_BATCH_SIZE`/`EMBEDDING_MAX_LENGTH`/`EMBEDDING_DIMENSION`/`EMBEDDING_CACHE_DIR` (unset ŌåÆ HF default cache). **Startup (decision 6):** when this provider + `EMBEDDING_DEVICE=cuda` are selected, the FastAPI app **loads the model once at startup and fails fast** if CUDA/model load fails (`validate_startup`); every other provider (and non-vector app roles ŌĆö ingest/review/lint) skip warmup entirely and stay GPU-independent. Validate with `scripts/check_embedding.py`. Tests split a **Torch-free unit layer** (always run) from **opt-in real-model tests** (`@pytest.mark.model` CPU, `@pytest.mark.gpu` CUDA; gated behind `KS_RUN_EMBED_TESTS=1`).

**Vector Index**
The derived **LanceDB** index under `indexes/vector/` that embeds the *same* per-source chunks as the keyword evidence index. Each row carries the embedding plus citation metadata identical to a keyword evidence hit (authoritative anchor `(source_id, char_start, char_end)`), so a vector hit returns the same [[structured-citation]] object and "joins the same `/search` contract without changing the shape." It is gitignored and regenerable (ADR-0032 ┬¦7), backed up only on opt-in, and refreshed **explicitly** (`scripts/reindex_vector.py`, never the per-file hook) ŌĆö the keyword channel stays the always-fresh path while the vector index may drift between reindexes by design (ADR-0033).

**Embedding Identity (Staleness Key)**
The contract that decides when the vector index must be rebuilt. `embedding_model_ref` **is** the embedding identity ŌĆö the operator pins it precisely and bumps it whenever the served model, quantization, pooling, normalization, or version changes. The **index-level** key (`embedding_model_ref`, `embedding_code_version`, `distance_metric`, `dimension`, `index_schema_version`) and **per-row** key (`source_id`, `chunk_id`, `chunk_fingerprint`, `embedding_model_ref`) drive rebuilds: any index-level mismatch refuses an incremental run and requires `--force`; a chunk-fingerprint-only diff re-embeds just the changed chunks. Bit-identical vectors are **not** required (cosine ranking tolerates GPU float nondeterminism); runtime guards validate the returned dimension and (HTTP seam only) cross-check the server's `model` against the ref (ADR-0033). For the [[in-process-flagembedding-backend]] the ref is derived torch-free as `flagembedding_bge_m3:<EMBEDDING_MODEL_ID>:<fp16|fp32>` (`resolve_model_ref`; precision is folded in), so switching off the TEI seam ŌĆö or flipping fp16 ŌĆö forces a `--force` rebuild (ADR-0053).

**Summary-First Navigation**
The rule that every major wiki page includes a short `> [!summary]` callout so humans and agents can decide whether to open the full page.

**System of Record**
The canonical project repository under WSL at `~/code/knowledge-system`.

**Maintenance Pass**
A Phase 7 deterministic, job-recorded maintenance operation (`/jobs/lint`, `/jobs/reindex`, `/jobs/stale-check`, eval) that **detects and proposes**: it surfaces health problems, appends `wiki/log.md`, and files review items for anything semantic/destructive, acting autonomously only on safe, non-destructive operations (reindex, backup, candidate detection). There is **no scheduler/daemon** ŌĆö cadence is an OS `cron` recipe the operator opts into (ADR-0036, ADR-0004). "Failing lint" is an *outcome report*, not an aborted run (distinct from a deterministic `validate_*` integrity hard-fail). The LLM/heuristics propose; a human disposes; `/reviews/apply` actions the approved, executor-backed subset.

**Governance decision vs maintenance task**
The boundary that decides whether a lint finding files a **review item** or is **report-only** (ADR-0037). A *governance decision* asks a human to choose semantic truth or approve a destructive/irreversible action (deprecate, archive, contradiction resolution) ŌåÆ it belongs in the `reviews/` ledger. A *maintenance task* is a health defect with a **mechanical, non-decisional remedy** (re-run a producer) ŌåÆ it is report-only (structured findings + `by_check` counts, no ledger entry). Routing maintenance tasks through the ledger would blur the two and is treated as architectural drift.

**Summary rot** ([[lint-quality-heuristics]], ADR-0037)
A report-only `/jobs/lint` finding (severity **low**): an *enriched* Source summary whose enrichment artifact (`normalized/enrichment/<sid>.json`) `input_fingerprint` no longer matches `artifact_fingerprint(current normalized markdown, current configured summary model_ref)` ŌĆö i.e. the current enrich pass *would regenerate it* (content/model/prompt/schema drift). A `summary_status: stub` or missing artifact is **not** rot (ADR-0016). Source pages only; remediation code `rerun_enrich`. Concept/Entity/Synthesis summary rot is deferred.

**Synthesis rot** (`synthesis_rot`, ADR-0037 decision 6)
A report-only `/jobs/lint` finding (severity **low**, graph-gated): an *active* synthesis whose durable artifact (`normalized/enrichment/<topic_id>.synthesis.json`) `input_fingerprint` no longer matches `synthesis._fingerprint(current topic, enrich_model_heavy)` ŌĆö i.e. the topic's evidence (claims/citations/disagreements) drifted since approval. This *is* the synthesis producer's existing `stale_active` signal, surfaced by lint key-free via topic-driven enumeration (`eligible_topics`). `subject` is the `synthesis_id`; remediation code `rerun_synthesis`. An active synthesis whose artifact is missing/unreadable ŌåÆ `synthesis_unverifiable` (degraded). **Evidence-gone** (a topic that no longer reconstructs because active evidence collapsed) is *not* rot and *not* degraded ŌĆö it is a verifiable governance-lifecycle condition owned by the synthesis producer's deprecation flow. **Concept/Entity summary rot is intentionally absent**: those pages are `generation_status: deterministic` graph projections with no LLM-authored summary, already enforced by [[validate_projection]] (a hard validator); a rot heuristic would duplicate it.

**Stale-claim citation** (`stale_claim_citation`, ADR-0037)
A report-only `/jobs/lint` finding (severity **medium**): a claim's cited anchor no longer supports the **stored** quote. For each durable `normalized/enrichment/<sid>.claims.json` citation that exactly matches an **active graph `derived_from` edge** on `(claim_id, source_id, char_start, char_end)`, the stored quote is re-grounded against the current normalized Markdown (`ground_citation(..., require_quote=True)`); failure ŌåÆ stale. It asserts the *anchor* is stale, **not** that the claim is false. Distinct from `uncited_claim` (a claim with no active source). Remediation code `rerun_extract_claims`. Graph-gated.

**Tier-2 extraction contract** (ADR-0055)
The explicit behavior contract the tier-2 concept/entity prompt (`_CONCEPTS_SYSTEM`) states, added after UAT showed a real model returning `concepts: []` on concept-rich documents while flooding `entities` with bibliography/byline names. Two halves: **concept elicitation** ŌĆö central recurring ideas/frameworks/themes/processes/methods/problems/trade-offs, typically 3ŌĆō10, most-central first, canonical form; an empty list is legitimate only for genuinely concept-free inputs (receipts, OCR noise, table dumps, short administrative records); never invent to satisfy a count; no named people/orgs/projects/products in `concepts`; abstractions allowed but document-supported. **Entity-noise boundary** ŌĆö provenance is not content: names appearing only in references/citations/bibliographies/footnotes/bylines/author lists/affiliations/acknowledgments/publisher metadata are excluded; persons/orgs/projects/products are entities only when discussed, acting, affected, compared, evaluated, quoted, or central to the document's claims (a document's own authors qualify only if the body discusses them ŌĆö manifest provenance remains the promotion-independence authority, ADR-0018). Enforced by prompt-version bump (`enrich-concepts-prompt-v2`; fingerprint + cache-key change drive opt-in re-extraction), a prompt-contract pin test, and the [[concept-starvation]] guard. `CONCEPTS_SCHEMA` unchanged (`minItems` rejected ŌĆö it would force hallucination on degenerate inputs). *(ADR-0059 restates this contract over the single `items` array of the [[knowledge-item-taxonomy]] ŌĆö elicitation, noise boundaries, and never-invent carry over; the concept/entity two-array split is retired.)*

**Concept starvation** (`concept_starvation`, ADR-0055)
A deterministic, key-free, artifact-driven guard for the F1 failure mode (a substantive document extracting zero concepts). Predicate per source: `concept_count == 0 AND (entity_family_count >= 5 OR claim_count >= 1)`, read from `normalized/enrichment/<sid>.concepts.json` / `<sid>.claims.json` ŌĆö one claim proves semantic substance; many entities without concepts is the failure signature; a degenerate document (no entities, no claims) is not flagged. Lives at two layers: the `extract_concepts` job summary reports starved sources at run time (`concept_starved`), and a `/jobs/lint` check `concept_starvation` (ADR-0037 family: report-only, never flips `failing`, **not graph-gated**) files findings at severity **medium** with remediation code `rerun_extract_concepts`. It observes artifacts, never calls a model, so CI stays key-free; a real-model enrichment-quality eval lane is deferred. *(Superseded by [[topic-starvation]] under the item taxonomy, ADR-0059 ŌĆö the predicate's node families no longer exist post-restart.)*

**Document-complete extraction coverage** (ADR-0056)
The tier-2 coverage contract: every extractable body region of a source gets an opportunity to be seen by the model ŌĆö not necessarily that the entire document is sent in one prompt. Adopted after the 12k-char head bias (`max_chars=12000` in every prompt builder) left 10 of 23 substantive live-vault sources concept-starved despite the ADR-0055 v2 prompt. Differentiated per pass: concepts/entities get **one full-document call** (input ceiling `ENRICH_CONCEPT_INPUT_MAX_CHARS` ŌĆö renamed `ENRICH_ITEMS_INPUT_MAX_CHARS` by ADR-0059; the ADR-0055 concept band stays intact, and a new **entity soft band** joins the prompt contract ŌĆö typically up to ~25 central entities per document, more only when substantively central, not merely mentioned; prompt contract not schema validation, riding `enrich-concepts-prompt-v3`); claims get a **windowed map + deterministic merge** over [[claim-window]]s (claim output scales with document length, so windows bound both input and output per call). Tier-1 summary/tags remain head-biased by documented limitation. Claims **stage before replacing**: the full replacement claim set is collected and validated before any supersede ŌĆö a failed run preserves the existing claim layer (stale-but-visible over silently-absent; if the Markdown changed underneath, `validate_citations` fails loudly until `extract_claims` succeeds), reversing the earlier retract-first ordering. Empty Markdown is a deterministic complete replacement and may supersede to an empty set. The run summary distinguishes `replacement_not_applied`/`stale_claim_layer_preserved` from ordinary parse errors.

**Claim window** (`chunk-greedy-v1`, ADR-0056)
A greedy run of consecutive normalized chunks, ordered by ordinal, bounded by the actual full-Markdown span `last.char_end - first.char_start` (inter-chunk headings/blank lines count ŌĆö they are inside the window text `markdown[first.char_start:last.char_end]`). Chunks are never split: a single over-budget chunk becomes a counted singleton window (`claim_window_over_budget`). Quotes are located **inside the window text** and translated to full-document offsets by adding the window's `char_start` ŌĆö never first-match-searched over the whole Markdown (repeated phrases would misanchor). No overlap in v1: a seam-straddling claim is usually *missed* (accepted recall limitation), distinct from `claims_dropped_ungrounded` (emitted quotes that fail to ground). Window budget knob: `ENRICH_CLAIM_WINDOW_CHARS` (default 12000, positive-int-validated). Each call is framed "segment *i* of *N*" with the source title and local section context, the latter inside an explicitly-untrusted `<segment_metadata>` delimiter, entity-encoded (`&`,`<`,`>`) so a heading containing the literal closing tag cannot escape the container ŌĆö document-derived heading text is never instruction-adjacent prompt (`enrich-claims-prompt-v2`). Non-empty Markdown whose window plan comes out empty (missing/corrupt/anchorless chunk file = broken ADR-0012 mdŌåöchunks invariant) **fails closed**: no model call, no supersede, typed `window_planning_failed` error, old layer preserved ŌĆö there is no whole-document fallback.

**Strategy ref** (ADR-0056)
The explicit extraction-strategy component of tier-2 identity: `claims_strategy_ref = "chunk-greedy-v1:{ENRICH_CLAIM_WINDOW_CHARS}"`, `concepts_strategy_ref = "full-doc-v1:{ENRICH_CONCEPT_INPUT_MAX_CHARS}"` *(ADR-0059: now `items_strategy_ref` over the renamed `ENRICH_ITEMS_INPUT_MAX_CHARS`)*. Enters both the artifact freshness fingerprint and the [[response-cache]] key, composed alongside (not overloaded into) schema version, prompt version, model ref, and normalized Markdown ŌĆö implementation extends `LLMClient.parse()` with an explicit optional `strategy_ref` and `cache_key()` with a third version component, never hiding the ref inside prompt/schema version strings. Prompt wording can change without pretending the strategy changed, and strategy tuning restales without editing prompt constants. Both knobs are **cost-bearing semantic knobs**: changing one restales that pass vault-wide (ADR-0033 config-ref precedent), so the next producer run is a deliberate, billable re-extraction wave.

**Coverage truncated** (`coverage: truncated`, ADR-0056)
The honesty marker for the concepts pass (items pass from ADR-0059 on, knob renamed `ENRICH_ITEMS_INPUT_MAX_CHARS`): a document exceeding the cap is truncated to the cap and marked `coverage: truncated` in the durable artifact **and** job metadata (not only stdout), so "document-complete" claims stay auditable and a future ADR-0037-family report-only lint can consume the marker (deferred ŌĆö this slice ships markers only).

**Review reconciliation** (auto-withdrawal, ADR-0057)
The symmetric machine retraction of extraction-stale review items, closing the ADR-0055 ┬¦5 deferral. One shared decision function (called by `concepts._recompose_node` going forward AND by the key-free catch-up sweep `scripts/reconcile_reviews.py` ŌĆö never two interpretations) reconciles a node's **unresolved** (`pending`|`deferred`) items with current node state: node tombstoned ŌåÆ withdraw its `promote_candidate_node`; node resurrected (active mentions again) ŌåÆ withdraw its recompose-provenance `deprecate_wiki_page`. Four audited withdrawal reasons: `node_tombstoned_no_active_mentions`, `node_resurrected_active_mentions`, `node_already_active`, `node_missing_or_rekeyed`. **Authority is per-surface:** node status authority is page frontmatter (ADR-0030, graph = mirror) so every status-based reason requires graph == page (the recompose hook writes both in one pass ŌĆö agreement by construction); edges are graph-SoT (ADR-0029) so the resurrection premise reads active mentions alone; `node_missing_or_rekeyed` needs page-side corroboration (page absent or an identity tombstone `merged`/`rekeyed`) ŌĆö graph-missing alone never withdraws. Deprecate ownership keys on **stored provenance, never node state and never generalized prose**: `proposal.reason_code: "no_active_mentions"` (new, recompose-filed items) ŌĆö state-keying is proven wrong by lint's `under_supported_concept` deprecates (active nodes, always mentioned); the sweep alone also accepts the exact legacy constant `"no active source mentions remain"` as a one-time migration shim. **Same-subject ownership rule:** `review_id = hash(type|subject)` makes same-node deprecates from different producers collide into one item (first filer owns the stored reason) ŌĆö reconciliation only owns items whose stored `reason_code`/legacy constant is `no_active_mentions`; all other same-subject deprecations remain human/flat-queue decisions and are never rewritten or withdrawn. **The sweep is preflight-gated fail-closed** (any failure ŌåÆ exit non-zero, nothing withdrawn): graph DB exists, schema version matches, Ōēź1 node (an empty/wrong-root DB must never drain the queue), and the graphŌåöwiki projection is valid over every reviewed family node (mirror-without-page / page-without-mirror / status disagreement each refuse). Terminal-status files found in `pending/` are skipped-never-withdrawn; malformed files are counted (parse vs schema split) and the sweep continues. Approved/rejected items are immutable audit, never touched; withdrawn subjects may re-file later. This is the ADR-0050 audited-withdrawal class (machine retracting its own unresolved proposals), not a governance decision (ADR-0037 boundary).

**Per-source review flow** (ADR-0058)
A high-volume review **lens** over extraction-caused review items, grouped by source and reviewed sequentially; the flat `/ui/reviews` queue remains canonical and type-complete. Attribution: `promote_candidate_node` under every source with an active `mentions` edge (multi-source candidates appear on every mentioning source's screen with a badge; the **first decision resolves globally**, later screens render it read-only with decided-by/when; recurrence-eligible items labeled); `change_entity_subtype` ŌĆö `change_item_type` from ADR-0059 on ŌĆö under `context.source_id`; `deprecate_wiki_page` in a "Retired by re-extraction" section only when deterministic ŌĆö unresolved Ōł¦ recompose-provenance ([[review-reconciliation]] gate) Ōł¦ zero active mentions Ōł¦ `H == {S}` where `H` = distinct sources over the node's superseded mentions (`|H| ŌēĀ 1` stays flat-only). All other types stay global (governance/lifecycle/identity, not extraction cleanup). Sequence UX: source index in manifest ingest order (tie-break `source_id`), zero-item sources greyed as done, Start/Next ŌåÆ first source with remaining items, free jump, deferred never blocks. **Batch decide:** one HTML form per source, per-item approve/reject/defer radios defaulting to "no decision" (untouched = pending), one submit looping the existing single-item ledger primitives (no new primitive), already-decided rows skip-with-reason (never 409-abort). Apply is unchanged ŌĆö explicit, queue-wide, dry-run-previewed (ADR-0040). **Human-add producer path:** the reviewer adds a missed concept/entity; the POST performs producer-side writes immediately (candidate node + `asserted_by: human` anchorless mention ŌĆö a documented use of the nullable-anchor contract, no renderer pretends evidence text exists ŌĆö + candidate page) and files the promote item **pre-approved** (`decided_by: human`, the add is the approval), plus a purpose-named `audit_log/<id>-human-added-<hex>.json` entry; the normal executor promotes at apply. Duplicate routing: existing candidate ŌåÆ mention + reuse item; active ŌåÆ mention only; subtype collision ŌåÆ `change_entity_subtype`; cross-type ŌåÆ deferred; a terminal REJECTED promote slot **blocks the add** with a message naming the prior rejection and the explicit ADR-0045 reopen path (a human rejection is never silently reused, reopened as a side effect, or bypassed via a parallel subject). Guarded sweep shortcuts ("Approve N unchanged candidate nodes") are deferred with pinned constraints.

**Approve-with-amendments** (ADR-0058)
The optional amendments payload `POST /reviews/{id}/approve` accepts for `promote_candidate_node` only ŌĆö exactly `title`, `aliases`, `description`. Recorded **immutable** in the ledger at decision time; the **promote executor** applies at flip-to-active (single promotion writer, ADR-0018): node id stays frozen (never re-hashed, ADR-0021), slug re-derived from the amended title (executor owns the page move), old title auto-appended to aliases on change, aliases normalized/deduped page-authoritative, `description` = a new page-owned frontmatter field that recompose carries through re-extraction (like `split_from`). Reject rejects the candidate, not the amendment; **defer preserves typed amendments** in a mutable `draft_amendments` block inside the pending item file (excluded from identity ŌĆö `review_id` hashes only `type|subject`; never read by executors; frozen into the immutable payload at approve, discarded on reject). Subtype is NOT amendable ŌĆö routed to `change_entity_subtype` (rekey = the ADR-0041 bright line); conceptŌåöentity cross-type deferred. *(ADR-0059 revises this: `item_type` joins the amendable fields ŌĆö retype is a non-rekeying metadata flip under the type-neutral id, and it is REQUIRED when approving a candidate carrying the [[unclassified-sentinel]].)* Named hazard: after a title amendment, future extractions of the NEW name hash to a different node id (alias divergence) ŌåÆ duplicate candidate, repaired via the existing merge executors; extractor alias resolution is out of scope.

**Knowledge-item taxonomy** (ADR-0059, taxonomy v2 2026-07-09)
The one-level, 15-type classification of [[knowledge-item]]s by **knowledge-object role** (never by grammatical/named-entity type): `domain`, `ai_topic_area`, `problem_risk`, `use_case`, `method_technique`, `architecture_pattern`, `technology_capability`, `model`, `model_family_architecture`, `product_tool_platform`, `data_ontology_asset`, `standard_protocol_interface`, `infrastructure_hardware`, `governance_regulation`, `provider_institution` (system values lowercase snake_case; TitleCase is display convention; UAT label revision 2026-07-09 renamed `sub_domain`ŌåÆ`ai_topic_area` and `ai_model_family`ŌåÆ`model_family_architecture`, semantics unchanged). Replaces Person/Organization/Project/Concept/Entity: `person` is provenance-only (manifest `provenance.author`/ŌĆ”, never an item, v1), `organization` ŌåÆ `provider_institution` (substantively-discussed actors only), `project` splits by role, `concept`/`entity` are deleted (the F5 misrouting boundary). `model` = named/branded (Claude, Qwen); `model_family_architecture` = generic class/approach (transformers, MoE) ŌĆö priority puts `model` first so branded families never fall through. Two prompt-level groups drive band guidance and [[topic-starvation]]: *thematic* (domain, ai_topic_area, problem_risk, use_case, method_technique, architecture_pattern, technology_capability, model_family_architecture, governance_regulation; ~3ŌĆō10 central) and *named* (the other six; only when substantively central, up to ~25). The user-supplied **15-step priority order** in the system prompt breaks ties (domain ŌåÆ model ŌåÆ ai_topic_area ŌåÆ architecture_pattern ŌåÆ model_family_architecture ŌåÆ method_technique ŌåÆ technology_capability ŌåÆ use_case ŌåÆ problem_risk ŌåÆ product_tool_platform ŌåÆ standard_protocol_interface ŌåÆ data_ontology_asset ŌåÆ governance_regulation ŌåÆ infrastructure_hardware ŌåÆ provider_institution), with a substrate carve-out so runtime/compute-substrate software (vLLM, CUDA, Kubernetes) stays `infrastructure_hardware` despite `product_tool_platform` ranking higher. Noise boundaries carried from ADR-0055: bibliography/byline/affiliation-only names never qualify; **named publications/papers/reports are never items** (`research_publication` was removed in v2 ŌĆö a publication enters the vault only as an ingested source, and the host document itself is never an item); benchmarks/metrics classify by role (`evaluation_benchmark_metric` also removed in v2). Taxonomy changes are ADR-gated (type list is not a config knob). **Where an older glossary entry or ADR conflicts, ADR-0059 is dominant for the post-restart vault.**

**Item type / `change_item_type`** (ADR-0059)
`item_type` is governed classification **metadata** on a knowledge item ŌĆö mutable, never identity (the type-neutral `itm_` id never encodes it). When a later extraction classifies an existing name under a different type, the mention routes to the existing node and a `change_item_type` review item is filed (`subject: {node_id, to_item_type}`); **nothing auto-retypes**. Approval applies a metadata flip ŌĆö frontmatter + graph nodes-table + audit; no id change, no page move, no edge re-point, no tombstone ŌĆö making it a **non-rekeying** executor by ADR-0041's own axis. Replaces ADR-0051's `change_entity_subtype` rekey for the item era; merge (ADR-0050) and split (ADR-0052) remain the only identity surgery, both now single ops over the item family (survivor/spin-off keep their own `item_type`).

**Unclassified sentinel** (`unclassified_review_required`, ADR-0059)
The QA-only escape hatch in the items extraction enum ŌĆö a sentinel, never a production category. Forcing a 15-way choice would hide model uncertainty as silent misclassification (the F5 signature), so the model may emit it; the worker still creates the full evidence trail (candidate `itm_` node + page + `mentions` edge + review item). Allowed states: `candidate` only ŌĆö `active` with the sentinel is a validator-forbidden state. Excluded from recurrence auto-promotion; a promote approval without a real `item_type` amendment is blocked (`missing_required_item_type`); never a production taxonomy group ŌĆö on `index.md`/Source pages sentinel items render only under the QA bucket "Unclassified (review required)", last (full listing + exact projection preserved); the human-add path requires a real type (sentinel is model-only).

**Topic starvation** (`topic_starved`, ADR-0059)
The redefinition of [[concept-starvation]] for the item taxonomy, preserving its intent (a substantive source produced no topic layer): `topic_starved = thematic_item_count == 0 AND (named_item_count >= 5 OR claim_count >= 1)`, counted over the taxonomy's thematic/named groups (the sentinel counts toward neither; threshold 5 stays a module constant). Same two layers as before: items-job summary flag + report-only `/jobs/lint` check (ADR-0037 family, never flips `failing`), remediation `rerun_extract_items`.

**Display alias / display label** (ADR-0060)
The Obsidian-readability projection over id-keyed wiki pages: every generated wikilink whose target has a resolvable **display label** must carry it as display text ŌĆö `[[Claims/clm_x|Readable claim title]]` ŌĆö and the id-titled families gain frontmatter for search. The label contract is **two-layer** (review round 1): the *frontmatter/search label* keeps the **full** sanitised title (`aliases: ["<full title>"]` on Source/Synthesis/Query pages ŌĆö no length truncation, so long titles stay quick-switcher-matchable; Claims gain a display-only `title:` + single-entry `aliases:` derived deterministically from `claim_text` via `_claim_title` (Ōēż78, inherent to title derivation) ŌĆö `claim_text` stays the wording authority, ADR-0030), while the *rendered link label* in `[[target|label]]` position always comes from one shared `display_link_label(...)` helper (the `_claim_alias` discipline: de-linked, no `[ ] |`, collapsed, Ōēż78) so table cells and rows stay bounded. Labels resolve page-locally from frontmatter (`title:`; claims fall back to `claim_text`), never LLM or graph reads. **Filenames stay id-keyed permanently** (renames rejected ŌĆö citation paths, review subjects, graph slugs; Obsidian explorer/tabs/graph still show ids, a documented limitation). Enforcement splits by the validator/lint boundary: a blocking **alias-shape validator** (`scripts/validate_link_aliases.py`, separate from [[validate_wikilinks]] ŌĆö alias present and non-empty wherever a label resolves; bare links legal only when no label resolves; dangling targets stay validate_wikilinks' failure) over generated page bodies + `index.md`, while drift is only the report-only lint `display_alias_rot` (ADR-0037 family, severity low, remediation = the free re-render chain), which compares against `display_link_label(current title)` ŌĆö never the raw `title:` ŌĆö so a retitle never hard-fails referrers and long titles never false-positive against their capped link aliases. Display metadata only: never changes `answer_eligible`, citations, graph edges, or retrieval eligibility. Renderers stay IO-free (label maps passed in); alias text enters `input_fingerprint` as ordinary page bytes.

**Retention Lifecycle**
The reversible status path a source/page moves along under Phase 7 retention (`policies/retention.yaml`, Build Spec ┬¦12): `active ŌåÆ stale_candidate / archive_candidate ŌåÆ archived` (and the human-only `delete_candidate ŌåÆ deleted`). **Archive is a reversible status transition** applied to the manifest + Source page + graph node mirror, then reindex ŌĆö **never a physical move, rewrite, or deletion of raw bytes** (the locked Phase 7 invariant). Archived material **stays indexed** but is excluded from default retrieval/navigation by the existing status filter (`search.RETENTION_DEFAULT_STATUSES`, ADR-0032 ┬¦8); an explicit `source_status=archived` still finds it. The reversible transition is the **`archive_source`** review type (renamed from the misleading `archive_raw_file`), executor-backed in Phase 7 (`apply_archive_sources` flips `active ŌåÆ archive_candidate` ŌĆö the honest v1 terminal status, never `archived`/physically moved); `delete_raw_file` stays **record-only forever** (proposed/logged/decidable, executed only manually). The **durable Source lifecycle authority is `manifest["status"]`** (default `active`, written only by `manifests.set_status`): the Source page reads it and is a pure projection, so status flows `manifest ŌåÆ Source page ŌåÆ keyword nav index ŌåÆ retrieval filter` and survives a wiki regen (it is *not* page-preserved). `retention_class` (policy category) is distinct from `status` (lifecycle/retrieval visibility). **`missing_raw_source`** is the record-only review type governing a broken source record (a catalogued raw file gone missing ŌĆö a high-severity lint finding, never auto-remediated, since it may be a transient mount/drive failure). **`purge_response_cache`** is a single aggregate, record-only review type (stable subject `{"scope": "response_cache"}`) proposing a manual purge of the LLM [[response-cache]] when it exceeds the policy `cache_ttl_days`/`cache_max_mb` ŌĆö it has **no executor** (a bulk purge forfeits reproducibility, ADR-0027), and carries only counts/sizes/age, never cached responses or keys.
